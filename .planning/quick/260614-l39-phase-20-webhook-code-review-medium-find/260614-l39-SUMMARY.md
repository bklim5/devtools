### Finding 8 — root tsconfig `"types": []` (tsconfig.json)
Added `"types": []` to root `compilerOptions`. The `server/webhook` workspace member hoists `@types/node`, which was leaking Node globals (`process`/`Buffer`/`NodeJS.Timeout`) into the webview `tsc --noEmit` gate and masking browser-only type errors.

**Type errors surfaced: NONE.** Adding `"types": []` produced ZERO fallout — `pnpm tsc --noEmit` stayed green on the first run. The codebase was already browser-correct: call sites use `window.setTimeout` / `ReturnType<typeof setTimeout>` (both resolve to the DOM `number` once Node types are gone), and `src/lib/release/publishPlan.ts` deliberately avoids the `NodeJS` namespace. No `@types/node` re-added; no `@ts-expect-error`; `server/webhook/tsconfig.json` keeps `"types": ["node"]` untouched. Both gates green (webview tsc + server tsc) and vitest 889/889. (T-20-13)

### Finding 10 — non-vacuous Buy e2e (test/e2e/license-buy.e2e.ts)
The old spec asserted only hash-unchanged + modal-still-mounted — a silently-broken onClick satisfied both.

**Stub approach:** the app's `platform` singleton is module-private (unreachable from a `browser.execute` context), so the unit-test `setPlatformForTest` pattern is not available in-webview. The fallback path from the plan was used: wrap `window.__TAURI_INTERNALS__.invoke`. The opener plugin calls `invoke('plugin:opener|open_url', { url, with })` (confirmed in `node_modules/@tauri-apps/plugin-opener/dist-js/index.js`), so the recorder:
- captures `args.url` for the `plugin:opener|open_url` command into `window.__openUrlCalls`,
- SHORT-CIRCUITS only that command to `Promise.resolve()` (so no real OS browser opens during the WDIO run),
- delegates every other command to the stashed original invoke,
- is idempotent (a retry-loop double-install is a no-op, preserving the original).

New load-bearing assertion after `clickBuy()`: `openUrl` was called **exactly once with `https://tinkerdev.io/buy`**. The hash-unchanged + modal-mounted checks stay as corroboration. The wrapped invoke is restored in `finally` (alongside the existing toggle-back-to-FULL cleanup) so it never leaks into later specs. File header updated to describe the new contract; the native OS browser-open stays a MANUAL walkthrough item. (T-20-15)
