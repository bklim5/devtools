**WebKit lessons to consolidate into helpers.ts doc comments (sources: HARNESS.md +
sidebar.e2e.ts comments):**
1. macOS WebKit's embedded WebDriver (605.1.15) drops the Alt modifier on
   `browser.keys()`/W3C Actions — keydown arrives `altKey:false`, so altKey-guarded
   handlers correctly ignore it (RESEARCH.md:499). Hence bubbling-KeyboardEvent
   dispatch on `document.activeElement` (React listens at the document root via
   native bubbling — same real code path). → `dispatchKey` doc comment.
2. macOS Option+letter COMPOSES to a glyph (Option+P → "π"): app code must match
   physical `e.code === "KeyP"`, never `e.key`; e2e must dispatch the composed shape
   (`key: "π", code: "KeyP", altKey: true`) or it false-positives (D-17). → `dispatchAltP` doc comment.
3. WebKit's embedded WebDriver goes stale on CHAINED element handles across
   navigation/re-render — read DOM state in a SINGLE `browser.execute` round-trip
   (the url.e2e lesson). → doc comment on the DOM-reader helpers.

**browser.execute closure gotcha:** the executed function is serialized — it cannot
capture outer variables. Pass values as trailing args, e.g.
`browser.execute((h: string) => { window.location.hash = h; }, "#/tools/" + toolId)`.
The existing inline navigations hardcode the literal; the helper must pass the id as
an arg.
