# Phase 18 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 18's user-visible surfaces are (1) the shared `UpsellPanel` (D-19..D-22), (2) lock badges in sidebar + ⌘K palette rows (D-23..D-25, dormant in production per D-18 — exercised via fixture tests + dev toggle), (3) the locked treatment of reorder/pin affordances (D-28), (4) the sidebar-footer "Unlock Pro" row (D-29), and (5) the DEV-only palette toggle command (D-32). Shipped packaged behavior is visually UNCHANGED this phase (everything-unlocked in-Tauri default).
>
> **Canonical visual authority:** `design/DevTools Mockup.html` + the `@theme` tokens in `src/index.css`. This contract composes with those tokens; it does not introduce new ones.

## Spacing Scale

Declared values (Tailwind default 4px-step utilities; all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (`gap-1`), badge inset, compact button padding-y (`py-1`) |
| sm | 8px | Button padding-x (`px-2`), gap between paired buttons, sidebar row padding |
| md | 16px | Spacing between UpsellPanel elements (heading → body → CTA row) |
| lg | 24px | UpsellPanel internal padding (`p-6`) |
| xl | 32px | UpsellPanel max content inset from `<main>` edges when route-rendered |
| 2xl | 48px | Vertical centering offset for route-rendered panel (visual center, not exact) |
| 3xl | 64px | not used this phase |

Exceptions (pre-existing, locked by D-17 / WCAG 2.5.8):
- **24×24px minimum interactive targets** for sidebar row controls (pin, lock-aware affordances) — matches the v1.5 widened targets.
- Existing compact sidebar row metrics (e.g. `pr-14` control-zone reserve, `right-8`/`right-1` slots) are inherited unchanged; the lock badge goes **inline after the tool name** (status-badge slot family), NOT in the right control zone (RESEARCH Pitfall 7).
- `rounded-[7px]` button radius and `rounded-[6px]` icon-button radius are the established house radii — reuse, do not invent new radii.

## Typography

Phase-scoped ramp drawn from the existing app scale (app uses 10.5–19px; new surfaces use exactly these four):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (panel copy, button labels) | 12px | 400 | 1.5 |
| Label (sidebar rows, palette rows, footer "Unlock Pro" row) | 13px | 400 | 1.4 (existing row metrics) |
| Meta (visually-hidden "— locked" suffix sizing N/A; small helper text under CTA if needed) | 11px | 400 | 1.5 |
| Heading (UpsellPanel title) | 16px | 600 | 1.2 |

Weights: exactly **400 (regular)** and **600 (semibold)** for all new Phase 18 surfaces. (The app uses 500 elsewhere; do not add it to new components — primary CTA emphasis comes from the accent-soft fill, not weight.)

Font: `font-sans` (IBM Plex Sans) for everything this phase. No mono surfaces.

## Color

All values are existing `@theme` tokens from `src/index.css` — no new colors may be introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg-app` #0a0b0d / `--color-pane` #14161b | Window background, `<main>` pane behind the route-rendered UpsellPanel |
| Secondary (30%) | `--color-sidebar` #101216, `--color-card`/`--color-panel` #181b21, `--color-palette` #1a1d23 | Sidebar, UpsellPanel card surface, modal dialog surface; borders `--color-bd` rgba(255,255,255,0.07) |
| Accent (10%) | `--color-accent` #5b9bf8 (+ derived `accent-soft` 15% tint, `accent-line` 75%) | See reserved-for list below — NOTHING else |
| Destructive | `--color-bad` #f0876b | NOT used this phase (no destructive actions) |

Accent reserved for (explicit, locked by D-24 + app-wide accent-is-selection-only discipline):
1. Selected/active sidebar + palette row (existing, unchanged).
2. `focus-visible:ring-2 ring-accent` focus rings on every new interactive element (house style).
3. The UpsellPanel **primary "Buy license" CTA only**, using the existing UpdateBanner primary-button family: `border-accent-line bg-accent-soft text-accent` — never a solid accent fill.

Accent explicitly FORBIDDEN on:
- The lock badge — **neutral `text-tx-2` (#989da7) only**, never amber, never accent (D-24).
- The footer "Unlock Pro" row — quiet neutral row (`text-tx-2`, hover `text-tx`), D-29 "small, quiet".
- Locked reorder/pin affordances — they keep their existing neutral treatment; locked state adds NO color change (lock is conveyed by behavior + the panel, and by badge/SR name where applicable). **No opacity-only locked state, no dimming** (ENT-04).

Text contrast floor: body text uses `text-tx` (#e7e9ee) or `text-tx-2` (#989da7) on card/pane surfaces — both ≥4.5:1 (tx-2 ≈ 7:1 on #181b21). Never use `tx-3` below 11px.

## Component Contract (phase-specific)

### UpsellPanel (D-19..D-22 — layout FINAL this phase)
- ONE shared component, props: `{ feature: string, icon: LucideIcon }`.
- Surface: `bg-panel` card, `border border-bd`, `rounded-[7px]`, `p-6`, max-width ~420px.
- Structure top-to-bottom (16px gaps): feature icon (`text-tx-2`, 20px) + heading row → 1–2 line body (`text-[12px] text-tx-2`, no pricing) → CTA row.
- CTA row: primary "Buy license" `<button>` (UpdateBanner primary family, reads single exported `BUY_LICENSE_URL` constant, stubbed no-op this phase) + secondary "I have a license key" `<button>` (UpdateBanner secondary family: `border-bd bg-input-bg text-tx-2`, inert stub this phase). 8px gap between buttons.
- Two render surfaces, one component:
  - **Route placement** (D-30): rendered centered in `<main>` in place of a locked tool's UI.
  - **Modal dialog** (app-level locks: D-28 affordances + D-29 footer row): `role="dialog"` `aria-modal="true"` with `aria-labelledby` on the heading, `--color-scrim` backdrop, Esc + scrim-click dismiss, focus moved to the panel on open and **returned to the invoking control on close** — reuse the ⌘K palette's scrim/dismiss/focus patterns.
- WCAG-AA: real `<h2>` (or `<h1>` in route context) heading, visible `focus-visible` rings, all buttons Tab-reachable, ≥4.5:1 text.

### Lock badge (D-23..D-25 — dormant in production)
- lucide `Lock`, 12px, `text-tx-2`, `aria-hidden="true"`, placed **inline immediately after the tool name** inside the truncating span (sidebar) and after the name in palette rows.
- Accessible name: append visually-hidden ` — locked` text inside the NavLink/palette button (row reads "Protobuf Decoder — locked"). No `aria-live` for static lock state.
- Selecting a locked tool still navigates; the route shows the UpsellPanel — never hidden, never redirected.

### Locked affordances (D-28) + footer row (D-29)
- Locked pin/reorder controls stay fully visible with their existing neutral styling; pointer or keyboard invocation (Alt+↑/↓, Alt+P, Shift+F10 reset) opens the UpsellPanel modal instead of mutating prefs. Keyboard chords keep the `e.code` physical-key discipline (macOS Option-compose — RESEARCH Pitfall 9).
- Footer "Unlock Pro": a `<button>` (not NavLink) row after the `<nav>` inside the `<aside>`, bottom-anchored; `text-[13px] text-tx-2`, hover `text-tx`, `focus-visible:ring-2 ring-accent`, 24px min height; lucide `Lock` 12px leading glyph permitted (neutral). Shown only in free tier.

### Lazy-load state (ENT-05)
- `<Suspense fallback={null}>` — no spinner, no skeleton (chunks load from local disk; a spinner would flash). If real-WKWebView verification shows a perceptible blank, fall back to a minimal neutral `bg-pane` placeholder — never a spinner.

## Interaction Contract Summary

| Surface | Trigger | Result |
|---------|---------|--------|
| Locked tool route | Navigate (sidebar click, palette select, URL) | UpsellPanel rendered in `<main>` in place of tool UI |
| Locked pin/reorder affordance | Pointer click, Alt+↑/↓, Alt+P, Shift+F10 reset | UpsellPanel modal opens; no prefs write |
| Footer "Unlock Pro" | Click / Enter / Space | UpsellPanel modal opens |
| UpsellPanel modal | Esc / scrim click | Dismiss; focus returns to invoker |
| Dev toggle (DEV builds only) | ⌘K → "Toggle free tier (dev)" | Writes downgrade-only override + `refreshEntitlements()`; ALL surfaces flip live (no reload) |
