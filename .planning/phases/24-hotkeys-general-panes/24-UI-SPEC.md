# Phase 24 — UI Design Contract

> Visual and interaction contract for **Hotkeys & General Panes (native-touching)**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase appends **two new panes** (Hotkeys, General) to the already-shipped Settings modal (`SETTINGS_PANES` append-only registry — `SettingsModal.tsx` stays byte-unchanged, D-23-10 precedent). It **invents no new visual system**: every token, class, and interaction pattern below derives from the canonical `design/DevTools Mockup.html` (mapped into `src/index.css` `@theme`) and from the already-shipped `AppearanceSettings` / `LicenseSettings` panes, the `SegmentedControl`, the sidebar nav-item states, and the `CommandPalette`. **Reuse over reinvention is mandatory** — the pane wrappers, header heading order, and helper-text scale are cloned from the Phase-23 `AppearanceSettings` pane verbatim.
>
> Source decisions: `24-CONTEXT.md` D-24-1..D-24-12 (LOCKED) + `24-RESEARCH.md` (HIGH-confidence, seam-verified). All "Claude's Discretion" items below are resolved consistent with the existing system and flagged `[discretion]`.

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** (no shadcn). **Zero new webview runtime/dev deps is a binding wedge constraint** — initializing shadcn would violate it. The ONLY new dependency in all of v1.7 is `tauri-plugin-autostart@2.5.1` (Rust crate + npm binding), the explicit scoped exception for launch-at-login (D-24-7), and it adds **no UI components** |
| Preset | not applicable |
| Component library | none — hand-rolled components styled with Tailwind v4 utilities. Reuses in-repo `AppearanceSettings`/`LicenseSettings` (pane wrapper blueprint), `SegmentedControl` (toggle/select control), sidebar nav-item state classes, `CopyButton` patterns |
| Styling | Tailwind v4 **CSS-first** (`@import "tailwindcss"` + `@theme` in `src/index.css`; **NO `tailwind.config.js`, NO `postcss.config`**). Tokens generate utilities like `bg-pane`, `bg-panel`, `text-tx-2`, `text-tx-3`, `border-bd`, `text-accent`, `bg-accent-soft`, `border-accent-line`, `ring-accent`, `text-bad`/`text-warn` |
| Icon library | `lucide-react` (1.17.0, already a dep). **Hotkeys pane glyph: `Keyboard`** `[discretion]` (the mockup's "Keyboard" nav label is illustrative; the shipped pane is named "Hotkeys" per ROADMAP). **General pane glyph: `SlidersHorizontal`** `[discretion]`. Per-hotkey reset uses `RotateCcw`; the capture "recording" affordance may use no icon (text state) `[discretion]` |
| Font | Self-hosted **IBM Plex Sans** (body/UI) + **JetBrains Mono** (the captured accelerator chord display), vendored via `@fontsource` static weights — **zero runtime network / no CDN** |
| Routing | **HashRouter only**. Panes are append-only entries in the shell-level `SETTINGS_PANES` registry — NOT routes. The default-tool dropdown MUST NOT introduce any `location.assign`/path write; startup redirect stays `<Navigate>`-based (HashRouter-safe, D-24-10) |
| State pattern | All persisted state flows through the **single-writer prefs seam** (`updatePreferences` / `getSharedPreferences` / coercers in `prefsStore.ts`). **Never add a second prefs writer** (the exact Phase-23 data-loss bug — memory `prefs-blob-single-writer`). Gate any read of persisted chords/toggles on `prefsLoaded` / `ensurePreferencesLoaded` (memory `tauri-store-async-init-race`) |
| Theme | **Both light AND dark** (Phase 23 shipped live whole-app theming). Every surface/text/state below MUST clear WCAG-AA in BOTH themes — the `[data-theme="light"]` token block re-derives the same custom-property names, so utilities resolve correctly in both, but the planner must confirm no hardcoded dark-only color enters the new panes (Phase-23 Pitfall 5) |

## Spacing Scale

Declared values (multiples of 4; matches the shipped `AppearanceSettings` pane rhythm: `gap-6` wrapper, `gap-2` sections, `gap-1` header, `p-8` outer pad):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding (`gap-1` header) |
| sm | 8px | Section internal vertical rhythm (`gap-2`), button/control row gaps |
| md | 16px | Between-section vertical rhythm within a pane (`gap-4`); toggle-row internal padding family |
| lg | 24px | Pane wrapper vertical rhythm between blocks (`gap-6`, the `AppearanceSettings` wrapper value) |
| xl | 32px | Pane outer padding (`p-8` = 32px, matching `AppearanceSettings`/`LicenseSettings`) |
| 2xl | 48px | Reserved — major break, not expected on these panes |
| 3xl | 64px | Reserved — not expected on a single modal pane |

Exceptions:
- **Tap/focus targets ≥ 24×24px (WCAG 2.5.8)** — the capture field, each Reset control, the default-tool dropdown trigger, and each toggle control meet `min-h-6` / `h-6 w-6` (sidebar pin/grip precedent). The capture field is a full-width clickable control (comfortably ≥ 44px tall recommended `[discretion]`).
- **Control internal padding `px-4 py-2`** — reuse the `AppearanceSettings` segmented-control sample padding (`px-4 py-2 text-[13px]`); do not introduce a new control size.
- **No fixed widths** — panes are layout-agnostic (responsive Tailwind); the capture field and dropdown fill their column.

## Typography

Three sizes, two weights — **mirrors the shipped `AppearanceSettings` pane EXACTLY** (`text-[15px]` semibold pane header, `text-[13px]` medium section headings + body, `text-[12px]` helper). Line-heights: body 1.5, headings 1.2. No new sizes/weights beyond this ramp.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Pane header (h3) | 15px | 600 (semibold) | 1.2 | "Hotkeys" / "General" — `text-[15px] font-semibold text-tx`, one level under the dialog `<h2>` (preserves the Phase-22.1 heading order h2→h3→h4) |
| Pane header sub-text | 13px | 400 (regular) | 1.5 | "Rebind the app's keyboard shortcuts." / "App behavior and startup." — `text-[13px] text-tx-2` |
| Section heading (h4) | 13px | 500 (medium) | 1.2 | "Global summon" / "Command palette" / each General toggle group label — `text-[13px] font-medium text-tx` |
| Body / label / value | 13px | 400 (regular) | 1.5 | Toggle/row labels, dropdown option text, inline messages — `text-[13px]` |
| Helper / hint | 12px | 400 (regular) | 1.5 | Per-control helper sub-text, the "Esc cancels" capture hint, reserved-chord guidance — `text-[12px] text-tx-3` |
| Captured chord display | 12–13px | 500 (medium) | 1.5 | The resolved accelerator (`Cmd+Shift+D` form) shown in the capture field — **`font-mono`** (JetBrains Mono), keeps chord glyphs unambiguous |

**Constraint:** uses ONLY the `tx`/`tx-2`/`tx-3` color ramp + 400/500/600 weights already in the ramp. Mono (400/500) for the chord display only.

## Color

60 / 30 / 10 split, all tokens from `src/index.css` `@theme` (mapped from the canonical mockup), resolving in BOTH themes via the `[data-theme="light"]` re-declaration.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-pane` `#14161b` (dark) | The right content pane these two panes render into — the dominant surface |
| Secondary (30%) | `--color-panel` / `--color-card` `#181b21` (dark) | The capture field surface, each toggle row / control surface, the dropdown trigger surface |
| Accent (10%) | `--color-accent` `#5b9bf8` (+ derived `--color-accent-soft` 15%, `--color-accent-line` 75%) | **Capture field "recording" active state + every `focus-visible` ring + the selected/on state of a toggle** — nothing else (see reserved list) |
| Semantic WARN | `--color-warn` `#fbbf24` (+ `warn-soft`/`warn-line`) | **Optional** — only if the inline reject message is styled as a calm warning. Default treatment is neutral `text-tx-2` (see Copywriting). The reject is NOT an alarm |
| Borders | `--color-bd` `rgba(255,255,255,0.07)`, `--color-bd-2` `rgba(255,255,255,0.12)` | Capture-field border, toggle-row dividers, dropdown border |
| Input surface | `--color-input-bg` `#0d0f13` (dark) / `#f0f1f4` (light) | The capture field idle background + dropdown background — theme-aware, never a hardcoded dark hex (Phase-23 Pitfall 5) |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. The capture field's **"recording" active state** — `bg-accent-soft` fill + `border-accent-line` + `ring-accent`, signaling live key capture (D-24-1).
2. `focus-visible:ring-2 focus-visible:ring-accent` on EVERY focusable control (capture field, Reset buttons, dropdown, each toggle).
3. The **on** state of a boolean toggle — accent fill (matching the `SegmentedControl`/`AppearanceSettings` selected-segment `bg-accent-soft text-accent` precedent). The **off** state is neutral.

**Forbidden:** no alarm/red (`text-bad` `#f0876b`) for the reject message — a taken chord is a calm "try another", not an error alarm (D-24-2). `text-bad` is NOT used in these panes. The reject message default is neutral `text-tx-2`; `text-warn` is the only permitted escalation and is optional `[discretion]`.

## Copywriting Contract

All copy is calm/neutral (the app's house tone; D-24-2 mandates calm-not-alarm).

| Element | Copy |
|---------|------|
| Hotkeys pane header | `Hotkeys` |
| Hotkeys pane sub-text | `Rebind the app's keyboard shortcuts.` `[discretion]` |
| Summon row label | `Global summon` (helper: `Show TinkerDev from anywhere.`) `[discretion]` |
| Palette row label | `Command palette` (helper: `Open the ⌘K command palette.`) `[discretion]` |
| Capture field idle (with binding) | shows the resolved accelerator, e.g. `Cmd+Shift+D` (mono) |
| Capture field prompt (click to record) | accessible name `Rebind {Global summon}` — the field is a button; activating it enters recording `[discretion]` |
| Capture field recording state | `Press a shortcut…` (the live "recording" prompt; **`Esc` cancels** — D-24-1) |
| Capture cancel hint | `Esc to cancel` `[discretion]` (`text-[12px] text-tx-3`) |
| Per-hotkey Reset | accessible name `Reset {Global summon} to default` (`RotateCcw` icon button, keyboard-reachable — D-24-4) |
| **Inline reject — taken chord** | `That shortcut is already in use — try another.` (D-24-2 verbatim intent; neutral, no toast/modal) |
| **Inline reject — invalid chord** | `Add Cmd, Ctrl, or Alt to set a shortcut.` `[discretion]` (D-24-3: a chord must include a non-shift modifier) |
| **Inline reject — OS-reserved chord** | `That shortcut is reserved by macOS — try another.` `[discretion]` (D-24-3 blocklist) |
| **Inline reject — same-as-other** | `That shortcut is already used by {the other binding}.` `[discretion]` (prevent summon == palette) |
| General pane header | `General` |
| General pane sub-text | `App behavior and startup.` `[discretion]` |
| Launch-at-login toggle | `Launch at login` (helper: `Start TinkerDev automatically when you log in.`) `[discretion]` |
| Start-in-tray toggle | `Start in the menu bar` (helper: `Launch hidden; summon or click the tray icon to show.`) `[discretion]` |
| Default-tool dropdown label | `Open to` (first option literal `Last used`, then each enabled tool by name — D-24-10) |
| Show-license-in-sidebar toggle | `Show license status in sidebar` `[discretion]` |
| Active-change announcement (aria-live) | the rebind result is announced politely, e.g. `Global summon set to Cmd+Shift+D` (success) / the reject string above (failure) — `aria-live="polite"` (WCAG-AA, RESEARCH §9) |
| Empty state | **n/a** — both panes always render their full control set; no zero-data state |
| Error state | the inline reject messages above are the only "error" path — calm, inline, neutral, persist-nothing (D-24-2). No modal, no toast |
| Destructive confirmation | **none** — no destructive action in this phase. Reset-to-default restores a known-good value and is non-destructive (no confirmation needed) |

Tone: neutral, supportive. Capture/reset feedback goes through an `aria-live="polite"` region (not a toast).

## Layout Contract (append-only panes — D-23-10)

Both panes are appended to `SETTINGS_PANES` (`settingsPanes.tsx`); `SettingsModal.tsx` derives nav + content 1:1 with **zero shell change**. Each pane wrapper matches the `AppearanceSettings`/`LicenseSettings` convention:

- **Pane wrapper:** `flex flex-col gap-6 overflow-auto p-8` (no double-pad — the modal right-pane renders the pane content directly).
- **Header:** `<header className="flex flex-col gap-1">` → `<h3>` pane title + `<p className="text-[13px] text-tx-2">` sub-text.
- **Sections:** `flex flex-col gap-2` blocks, each with an `<h4>` section heading + optional `text-[12px] text-tx-3` helper.

### Hotkeys pane structure (top → bottom)

A list of **two binding rows** (summon, palette), each row showing: **label · helper · current chord (mono) · live-capture affordance · per-hotkey Reset** (D-24 "Specific Ideas"). `[discretion]` Recommended: ONE reusable `HotkeyCaptureField` component drives both rows (capture + reject + reset), since both share the identical capture/match contract (RESEARCH §4).

| Row part | Treatment |
|----------|-----------|
| Label + helper | `h4` label (`text-[13px] font-medium text-tx`) + `text-[12px] text-tx-3` helper |
| Capture field (idle) | a full-width button-like control on `bg-input-bg`, `border border-bd`, `rounded-[7px]` family, displaying the resolved accelerator in `font-mono`. `focus-visible:ring-accent`. ≥ 24px tall (44px recommended) |
| Capture field (recording) | `bg-accent-soft` + `border-accent-line` + `ring-accent`; text swaps to `Press a shortcut…`; window keydown captured from **physical `e.code` + modifier flags** (NEVER `e.key` — macOS Option+letter composes a glyph; D-24-1, Pitfall 2); `Esc` cancels |
| Reset control | small `RotateCcw` icon button, `text-tx-2 hover:text-tx`, `focus-visible:ring-accent`, accessible name `Reset {binding} to default`, ≥ 24×24px |
| Inline reject | neutral `text-[12px] text-tx-2` line directly below the field (calm, no toast/modal); cleared on the next valid capture |

### General pane structure (top → bottom)

Four control rows (final set, all ship — D-24-7..11):

| Control | Pattern |
|---------|---------|
| `Launch at login` | boolean toggle (accent-on / neutral-off). Seeds from `platform.autostart.isEnabled()` AND persists intent in prefs; on flip calls enable/disable AND `updatePreferences` (RESEARCH §7) |
| `Start in the menu bar` | boolean toggle (same pattern) |
| `Open to` (default tool) | a **dropdown / select** whose first option is literal `Last used`, then `ENABLED_TOOLS.map(t => t.name)` (D-24-10). Validate persisted `defaultToolId` via `getToolById` (untrusted-prefs coercion → fall back to "Last used"). `[discretion]` reuse `SegmentedControl` ONLY if the option count stays small; with 11 enabled tools a native-styled `<select>` / listbox is the better fit — must be keyboard-operable and `focus-visible:ring-accent` |
| `Show license status in sidebar` | boolean toggle (same pattern) |

**Toggle visual contract** `[discretion]`: a labeled switch or a two-segment `SegmentedControl` (On/Off) — must be keyboard-operable (Space/Enter), carry an accurate `aria-checked`/`aria-pressed` (or `role="switch"`), visible `focus-visible:ring-accent`, and **no opacity-only state** (on vs off distinguished by accent fill + position/label, not opacity). The off state is neutral, the on state uses `accent-soft`/`accent` per the `SegmentedControl` selected precedent.

## Keyboard & A11y Model (WCAG-AA — SET-08 / SET-09, RESEARCH §9)

| Concern | Contract |
|---------|----------|
| Pane reachability | Both panes reachable via the existing Settings nav (`aria-current` button-list, clamped Arrow/Home/End — unchanged shell). Tab order within a pane is DOM order: header → each section's controls |
| Capture affordance | The capture field is a real focusable control (button semantics); activating it (click OR Enter/Space) enters recording — **no mouse-only path** (D-24-5 / RESEARCH §9). While recording, a window-level keydown listener captures the chord; `Esc` cancels and returns the field to idle |
| Capture key source | physical `e.code` + modifier flags ONLY (never `e.key`); `e.preventDefault()` during recording so the chord doesn't trigger app shortcuts. One pure `keyEventToAccelerator(e)` helper feeds capture, persist, and the in-webview palette matcher (consistency — D-24-1/6) |
| Reject feedback | calm inline message + an `aria-live="polite"` announcement; prior binding stays active; persist nothing (D-24-2). No focus steal, no modal |
| Reset | each Reset is a keyboard-reachable button; activating it restores the shipped default and announces the result politely (D-24-4) |
| Toggles | each toggle is keyboard-operable with an accurate ARIA state (`role="switch"` + `aria-checked`, or a pressed button); visible focus ring; no opacity-only state |
| Default-tool select | keyboard-operable native select / listbox; the selected value is the accessible value |
| Contrast | every surface/text/state clears AA in BOTH themes (Phase-23 bar); the planner confirms no hardcoded dark-only color enters the panes (Pitfall 5) |
| Live region | a single polite `aria-live` region per pane carries rebind/reset results (not a toast) |

**Native-chrome caveat (RESEARCH Pitfall 6):** the actual OS register-failure, launch-at-login plist round-trip, and start-in-tray reveal/no-flash are NOT WebDriver-drivable — they are **human-walkthrough** items at the phase boundary plus a `gsd-ui-review` WCAG-AA audit. The webview-testable parts (capture field, palette-on-configured-chord, toggle visibility, prefs coercion) are unit + real-WKWebView e2e covered.
