| hand-edited prefs.json → app | Untrusted persisted blob; a user can inject a bogus chord / tool-id |
| webview → OS (autostart) | New native capability: launch-at-login registration |
| captured KeyboardEvent → accelerator string | Untrusted key input normalized to a persisted/registered chord |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Tampering | prefsStore chord coercers | mitigate | `isValidAccelerator(value)` gate per chord field; invalid → shipped default (D-24-12). Tested in prefsStore.test.ts. |
| T-24-02 | Tampering | prefsStore `coerceDefaultToolId` | mitigate | Validate via `getToolById` (ENABLED_TOOLS only) before persisting; unknown → null. Same guard resolveStartupTool already applies (T-02-07/08). |
| T-24-03 | Elevation | tauri-plugin-autostart registration | mitigate | Pass `None::<Vec<&str>>` launch args (no shell-injection surface); capability scoped to EXACTLY the three `autostart:allow-{enable,disable,is-enabled}` perms — no wider grant. **Scoped-exception rationale (D-24-7):** the only new dep in all of v1.7; official Tauri plugins-workspace package; ships no UI; reached only through `platform.autostart`; writes a per-user LaunchAgent plist, no network. |
