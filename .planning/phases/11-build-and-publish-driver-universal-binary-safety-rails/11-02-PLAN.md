<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| maintainer machine -> public releases repo | The irreversible publish: whatever lands becomes the auto-update every install trusts |
| build output dir -> latest.json -> updater | The `.sig` glob + signature copy: a wrong/stale file here ships an unverifiable update to everyone |
| process env -> tauri/gh subprocesses | Signing key + Apple creds inherited into child processes; never on argv, never logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-06 | Tampering | all CLI calls | mitigate | `execFileSync` argv arrays ONLY (no `execSync` string) — no shell-injection surface; mirrors Phase 10 T-10-04; grep asserts zero `execSync` |
| T-11-07 | Tampering | fresh-sig glob | mitigate | Glob ONLY `target/universal-apple-darwin/release/bundle/macos/*.app.tar.gz.sig`; clear prior `.sig` before build; `assertSingleSig` fail-on-0/>1 — a stale arm64 `.sig` can't ship (P1/P2) |
| T-11-08 | DoS / Tampering | gh publish ordering | mitigate | Assets (DMG + `.app.tar.gz`) via `gh release create` FIRST, `latest.json` via `gh release upload` LAST — the `latest` redirect never serves a manifest whose asset 404s (P3) |
| T-11-09 | DoS / Tampering | publish target repo | mitigate | Every `gh` call passes `--repo bklim5/devtools-releases`; never `origin`/private source — unauthenticated updater clients can fetch (P5); grep asserts the slug, never the private one |
| T-11-10 | Information disclosure | signing/Apple secrets | mitigate | Presence-checked booleans only; passed via inherited `{env: process.env}`; never in an argv array, never `log()`-ed a value (P6); grep audits no secret value logged |
| T-11-11 | Tampering | served-version drift | mitigate | Post-publish `curl -L` of the live endpoint -> `assertVersionMatches(servedVersion, cutVersion)` — a stale/wrong-version `latest.json` is caught before the manual gate (REL-12) |
| T-11-12 | DoS | single-arch build mislabeled | mitigate | `lipo -archs` on the real Mach-O must satisfy `parseLipoArchs` (both x86_64 + arm64) before publish — an arm64-only binary served as universal is rejected (P4/P7) |
</threat_model>
