<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS clipboard ↔ app | Clipboard read/write crosses into the app; scoped by Tauri capability permissions |
| network ↔ runtime | App is offline-by-design — any runtime network call (e.g. CDN font) is a boundary violation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information disclosure | Runtime CDN font fetch (fonts.googleapis.com) leaks usage/IP, violates offline-by-design | mitigate | Vendor fonts via @fontsource; acceptance criterion greps `dist`/`src`/`index.html` for googleapis/gstatic → must be absent (FND-05) |
| T-01-02 | Elevation of privilege | Over-broad Tauri capability grants more OS access than needed | mitigate | Grant ONLY `clipboard-manager:allow-read-text` + `allow-write-text` in capabilities/default.json (least privilege); no other plugin permissions in Phase 1 |
| T-01-03 | Tampering | Hostile pasted bytes drive the decoder into unbounded recursion/allocation | accept | Already bounded by the ported decoder's MAX_DEPTH / MAX_PAYLOAD_BYTES (verified by the 19 tests); not re-implemented here |
</threat_model>
