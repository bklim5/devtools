|----------|-------------|
| WebDriver server (:4445) ↔ network | The embedded W3C server is a remote-control surface; must exist ONLY in debug builds bound to localhost |
| automation harness ↔ app | The UI-gate driver becomes the trusted verifier for all later phases — its correctness gates everything downstream |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Elevation of privilege | WebDriver server (47 W3C endpoints) shipped in a release build = remote control of the app | mitigate | Plugin gated under `[target.'cfg(debug_assertions)'.dependencies]` + `#[cfg(debug_assertions)]`; cross-checked absent from Plan 03's release bundle (RESEARCH Pitfall 4) |
| T-01-11 | Spoofing | WebDriver server bound to a routable interface lets a remote actor drive the app | mitigate | Bound to `127.0.0.1:4445` (localhost only); never `0.0.0.0`; debug-only |
| T-01-12 | Repudiation | Automation path decision undocumented → later phases can't reproduce the UI gate | mitigate | Chosen path + rationale + exact commands recorded in docs/phase-0-notes.md (HRN-02 deliverable, D-03) |
</threat_model>
