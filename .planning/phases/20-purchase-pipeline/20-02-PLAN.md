Lemon Squeezy order_created webhook (D-60 verify):
- Header `X-Signature` = hex HMAC-SHA256 of the RAW request body with the store webhook secret.
- Verify with: `crypto.createHmac("sha256", LS_WEBHOOK_SECRET).update(rawBody).digest("hex")`
  then `crypto.timingSafeEqual(Buffer.from(digest,"utf8"), Buffer.from(sigHeader,"utf8"))`.
- Payload: `meta.event_name === "order_created"`; order id = `data.id`;
  buyer email = `data.attributes.user_email`. (Verify exact field paths against a real
  test-mode payload during Plan 03's D-63 e2e — log the raw event once. A5.)

Keygen CE (admin-token Bearer auth, JSON:API content-type `application/vnd.api+json`),
proven live in the Phase-19 SPIKE (scripts/keygen-ce/bootstrap.sh):
- Search (idempotency, D-58):
  `GET /v1/accounts/{acct}/licenses?metadata[orderId]={ORDER_ID}` → empty data[] ⇒ create.
- Create (D-58 metadata + policy rel; entitlements inherited from the policy, D-54):
  `POST /v1/accounts/{acct}/licenses`
  `{ "data": { "type":"licenses",
      "attributes": { "metadata": { "orderId": "{ORDER_ID}" } },
      "relationships": { "policy": { "data": { "type":"policies", "id":"{POLICY_ID}" } } } } }`
  → response `data.attributes.key` is the license key to email.
- Entitlements (pro.theming/pro.ordering) are attached to the POLICY in Plan 03's setup.sh,
  so they auto-propagate to every license — the backend does NOT attach per-license.

Resend (npm resend@6.12.4, D-64/D-66):
  `new Resend(RESEND_API_KEY).emails.send({ from:"TinkerDev Licenses <licenses@tinkerdev.io>",
    to:[buyerEmail], subject:"Your TinkerDev license key", text: <plain text> })`
  text = key + 3-line activation steps ("open TinkerDev → Unlock Pro → paste & Activate") +
  download link `https://github.com/bklim5/devtools-releases/releases/latest` + reply-for-help.

Fulfillment flow (D-58/D-59/D-60):
  POST webhook (RAW body) → verify sig (invalid ⇒ 401) → ignore non-order_created (200 ack)
  → extract orderId+email → search metadata[orderId] (exists ⇒ 200 idempotent skip)
  → create license (fail ⇒ 5xx, LS retries) → email key (fail ⇒ 5xx + alert) → 200.
