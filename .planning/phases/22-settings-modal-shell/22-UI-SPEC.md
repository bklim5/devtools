# Phase 22 — UI Design Contract

> Visual and interaction contract for **Settings Modal Shell, Entry Points & License Pane**. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase ships ONE new app-chrome surface — a **shell-level Settings modal** (Claude-style large centered dialog, paned layout) — plus four entry points (app menu ⌘, · tray · sidebar "Settings" row · ⌘K command) and the re-point of the Phase-21 footer/⌘K "License" affordance to open this modal on the License pane. It **invents no new visual system**: every token, class, and interaction pattern below derives from the canonical `design/DevTools Mockup.html` (mapped into `src/index.css` `@theme`) and from the already-shipped `UpsellModal`/`UpsellPanel`, `LicenseSettings`, `Sidebar`, and `CommandPalette`. **Reuse over reinvention is mandatory** — the modal a11y mechanics are cloned verbatim from `UpsellModal`, and `LicenseSettings.tsx` renders UNCHANGED inside the License pane (SET-06).
>
> Source decisions: `22-CONTEXT.md` D-S1..D-S11 (LOCKED). All "Claude's Discretion" items below are resolved consistent with the existing system and flagged `[discretion]`.

## Spacing Scale

Declared values (multiples of 4; matches the mockup's `--gap: 16px`, the window `13px` radius family, and the shipped `UpsellPanel` `gap-4`/`p-6` + `LicenseSettings` `gap-12`/`p-8` rhythm):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding (`gap-1`) |
| sm | 8px | Compact element spacing, nav-item internal gap, button row gaps (`gap-2`) |
| md | 16px | Default element spacing; nav-list item vertical rhythm; dialog header padding family |
| lg | 24px | Card inner padding (`p-6`), left-nav container padding (`p-[14px]`→16-ish, see exception), pane top padding |
| xl | 32px | Right content-pane outer padding (`p-8` = 32px, matching `LicenseSettings`) |
| 2xl | 48px | Major vertical break between blocks inside a pane (`gap-12`, reused from `LicenseSettings`) |
| 3xl | 64px | Reserved — not expected on a single modal pane |

Exceptions:
- **Left-nav container padding `p-[14px]` (14px)** — carried verbatim from the Sidebar (`p-[14px]`) for visual continuity between the app sidebar and the modal nav list. This is the established sidebar value, not a new token.
- **Nav-item / button focus + tap targets ≥ 24×24px** (WCAG 2.5.8) — the explicit close (×) control and each left-nav item meet `min-h-6`/`h-6 w-6`, matching the sidebar pin/grip precedent.
- **Modal max size caps** (layout-agnostic, no fixed pixel widths beyond caps — see Layout below): width `min(880px, 92vw)`, height `min(640px, 86vh)`. `[discretion]` — sits between the ⌘K palette cap (`min(560px,92vw)`) and the mockup `.window` cap (`min(1180px, …)`); "large centered" per D-S4, comfortably fits the 5-pane future without a fixed width.
- Action buttons reuse `px-3 py-1` (12px / 4px) from `UpsellPanel` `PRIMARY_BTN_CLASS`/`SECONDARY_BTN_CLASS` — do not introduce a new button size.

## Typography

Three sizes, two weights (mirrors the shipped `UpsellPanel`/`LicenseSettings`: 16px semibold heading + 13.5px/12px body; mono for key/email). Line-heights: body 1.5, heading 1.2 — the exact values in `HEADING_CLASS`/`BODY_CLASS`.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Heading (dialog title + card headings) | 16px | 600 (semibold) | 1.2 | Dialog title ("Settings") and the License-pane card headings — reuse `text-[16px] font-semibold leading-[1.2] text-tx` (`HEADING_CLASS`). The dialog title is the `aria-labelledby` target |
| Nav item label | 13.5px | 600 (semibold) | 1.2 | Left-nav pane labels ("License") — reuse the sidebar `text-[13.5px] font-semibold` row label |
| Body / label / value | 12px | 400 (regular) | 1.5 | Pane body text, field labels, status lines — `text-[12px] leading-[1.5]` (`BODY_CLASS`). Monospace (`font-mono`) for masked key + email (inside `LicenseSettings`, unchanged) |

Section/group eyebrow labels (e.g. an optional "PANES" cap on the nav list) reuse the established uppercase eyebrow: `text-[10.5px]`–`11px` semibold, `tracking-[0.09em]`, `uppercase`, `text-tx-3` (from the palette `.pglabel` / sidebar `pro-head`). `[discretion]` — with a single pane in Phase 22 the eyebrow may be omitted; if added it must use these exact values.

**Constraint:** no new font sizes/weights beyond the ramp above (`tx`/`tx-2`/`tx-3` color ramp, 400/600 weights only). The mono weights stay 400/500 as already vendored.

## Color

60 / 30 / 10 split, all tokens from `src/index.css` `@theme` (mapped from the canonical mockup). Dark theme only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-win` `#15171c` (dialog body) + `--color-pane` `#14161b` (right content pane) | The modal's dominant surfaces — body + content area |
| Secondary (30%) | `--color-sidebar` `#101216` (left nav list) + `--color-panel`/`--color-card` `#181b21` (cards inside the pane) | Nav list rail + card surfaces |
| Accent (10%) | `--color-accent` `#5b9bf8` (+ derived `--color-accent-soft` 15%, `--color-accent-line` 75%) | **Selected pane-list item ONLY** (left-nav active state) + focus-visible rings + the License-pane primary CTA fill — nothing else |
| Semantic OK | `--color-ok` `#34d399` | The existing License-pane OK dot for Pro-active states (inside `LicenseSettings`, unchanged) — never accent |
| Borders | `--color-bd` `rgba(255,255,255,0.07)`, `--color-bd-2` `rgba(255,255,255,0.12)` | Dialog border (`bd-2`), nav/pane dividers (`bd`) |
| Scrim | `--color-scrim` `rgba(6,7,9,0.66)` | Dimmed backdrop behind the centered modal (same scrim token + opacity as the ⌘K palette and `UpsellModal`) |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. The **selected** left-nav pane-list item — accent-soft fill + a 3px accent left-bar (`scale-y` 0→1), exactly the sidebar `.navitem.on` pattern. This is the **selected-only-accent** rule (D-03 carries over; D-S12 tone note below).
2. `focus-visible:ring-2 focus-visible:ring-accent` on every focusable control in the dialog (nav items, close ×, and the License-pane controls).
3. The License-pane primary CTA fill (`bg-accent-soft text-accent border-accent-line`) — inside `LicenseSettings`, unchanged.

**Forbidden:** no alarm/red colors anywhere in the shell (D-77/D-83 carry over). The only non-neutral, non-accent glyph is the existing `text-ok` dot inside `LicenseSettings`. Hover/default/focus nav-item states are neutral (`tx-2`/`tx` + `rgba(255,255,255,0.035)` hover fill); accent appears ONLY on the selected item.

## Layout Contract (D-S3 / D-S4 — paned modal)

**Modal shell (Claude #9 style, D-S4):**
- Fixed full-viewport scrim layer: `fixed inset-0 z-[60] flex items-center justify-center bg-scrim` — **z-[60]** (matches `UpsellModal`: must cover the shell's bottom-right overlay stack at z-50).
- Dialog: large **centered** card on `bg-win`, `border border-bd-2`, `rounded-[13px]` (the mockup `.window` radius), `overflow-hidden`, shadow `shadow-2xl` (or the mockup's `0 40px 90px -20px rgba(0,0,0,0.75)` family). Size caps: `w-[min(880px,92vw)] h-[min(640px,86vh)]` — responsive, no fixed pixel width (layout-agnostic wedge).
- `role="dialog"` `aria-modal="true"` `aria-labelledby={dialogTitleId}` `tabIndex={-1}` on the dialog container (cloned from `UpsellModal`).
- **Dismissal (D-S5):** Esc (keydown), backdrop click (`onMouseDown` on the scrim where `e.target === e.currentTarget`), AND an explicit close (×) control top-right of the dialog header. All three call `closeSettings()`.

**Dialog internal structure (top → bottom):**
1. **Header bar** — dialog title (`Settings`, the `aria-labelledby` target) on the left, close (×) button on the right. Border-bottom `border-bd`. Padding `px-6 py-4` family. The × is a 24×24 neutral icon button (`text-tx-2 hover:text-tx`, `focus-visible:ring-accent`).
2. **Body** — two-column flex, `min-h-0` so the right pane scrolls independently:
   - **Left nav list** (`bg-sidebar`, `border-r border-bd`, `p-[14px]`, fixed-ish rail). A vertical list of pane items rendered from the pane registry. In Phase 22 it shows ONLY `License`. Width caps at the sidebar value family (`w-[200px]`–`w-[220px]` `[discretion]`, not full sidebar 268px — the modal is narrower) — responsive, not a hard pixel lock.
   - **Right content pane** (`bg-pane`, `flex-1`, `overflow-auto`, `p-8`). Renders the active pane. For License it renders `<LicenseSettings />` UNCHANGED.

**Left-nav item states (the pane-list, mirrors sidebar `.navitem`):**
| State | Treatment |
|-------|-----------|
| Default | `text-tx-2`, transparent bg, gear/pane icon `text-tx-2`, radius `rounded-[9px]`, padding `py-2 px-[11px]` |
| Hover | `hover:bg-[rgba(255,255,255,0.035)]` (sidebar hover fill) — neutral, no accent |
| Focus-visible | `focus-visible:ring-2 focus-visible:ring-accent focus-visible:ring-offset-0` |
| **Selected** (active pane) | `bg-accent-soft text-tx` + a 3px accent left-bar (`absolute left-[3px] h-[56%] w-[3px] bg-accent`, `scale-y-100`), icon `text-accent` — the **only** accent use in the nav (selected-only rule). Carries `aria-current="page"` OR `aria-selected="true"` (see Keyboard Model) |

**Right pane scroll behavior:** the right pane is the ONLY scroll container (`overflow-auto`); the left nav and header stay fixed. `LicenseSettings` already owns its own internal `overflow-auto p-8` — when hosted in the pane, the pane wrapper must NOT double-pad: render `LicenseSettings` directly as the pane content (it provides `p-8` + `gap-12`). `[discretion]` Heading reconciliation: `LicenseSettings` carries an `<h1 className="sr-only">License</h1>` and per-card `<h2>` headings; the dialog title `<h2 id={dialogTitleId}>Settings</h2>` (or `h1`) sits in the header. Keep the License `sr-only` h1 (the pane's own landmark) — no visible duplicate "License" title is needed because the selected nav item already names it; do NOT modify `LicenseSettings` to add one (SET-06 unchanged mandate).

## Keyboard & A11y Model (WCAG-AA — SET-04 / SET-05)

Cloned from `UpsellModal` for the trap/return mechanics, extended with arrow nav for the pane list.

| Concern | Contract |
|---------|----------|
| Focus on open | On mount the dialog container `.focus()`s (matches `UpsellModal`). `[discretion]` first Tab lands on the close × or the active nav item — both are inside the trap; recommend initial focus stays on the dialog, first nav item is the first focusable |
| Focus trap | Tab/Shift+Tab cycle within the dialog only — verbatim port of `UpsellModal`'s Tab-wrap handler (query `button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])`; wrap at both ends; pull focus back if it escapes). `aria-modal="true"` promises the background is inert |
| Return focus | On close, focus returns to the invoker captured **synchronously** at `openSettings()` time (the Phase-21 store fix). Transient openers (⌘K command, tray/menu event) pass an explicit persistent return target; falls back to `document.activeElement` then the dialog |
| Esc | Closes the modal (`closeSettings()`), highest-priority key |
| Pane-list arrow nav | `↑`/`↓` move selection between pane-list items; `Home`/`End` jump to first/last. `[discretion]` — follows the sidebar's `resolveRovingTarget` clamp model (no wrap). Arrow keys change the ACTIVE PANE (not just focus) so the right pane updates — this is the SET-05 "move between panes by keyboard". With one pane in Phase 22 arrows are no-ops but the handler ships extensible |
| Pane-list semantics | The nav list is a `role="tablist"`-style or `role="listbox"`/`aria-current` pattern. **Decision `[discretion]`:** use a vertical list where each item is a `button`; the active item carries `aria-current="page"`. Active-pane change is announced via a visually-hidden `aria-live="polite"` region (`{pane} settings`) — the SET-05 "active pane announced via aria" requirement. (A `tablist`/`tab`/`tabpanel` triple is an acceptable alternative; if used, the right pane is the `tabpanel` with `aria-labelledby` the active tab and `tabIndex={0}`. Pick ONE and keep it consistent for Phases 23–25.) |
| Tab order within dialog | header close × → left-nav items → right-pane focusables (the License-pane controls), then wrap. Backdrop is not focusable |
| Backdrop click | `onMouseDown` on the scrim closes; `onMouseDown` on the dialog `stopPropagation` (so a drag that starts inside and releases on the scrim does not close) — verbatim from `UpsellModal` |
| Contrast | All text/affordances meet AA on their surface (the `@theme` ramp is already AA-tuned: `tx-3` raised to `#868b95`, accent brightened to `#5b9bf8`). No opacity-only state (ENT-04 carries over) |

**Deep-link (D-S6, HashRouter):** `#/settings/license` must open the Settings modal on the License pane. Recommended wiring `[discretion]`: a small effect at the shell reads the hash route and calls `openSettings('license')` (the modal is shell-level, not a router element); the legacy `#/settings/license` route entry can map to a redirect-to-tool + `openSettings('license')` so existing links/e2e hash navigation still land on the License pane. Keep the modal the single surface (no duplicate in-window route render).

## License Pane Framing (SET-06 — D-S6)

- Render `<LicenseSettings />` **UNCHANGED** as the right content pane for the `license` pane id. Do NOT modify `LicenseSettings.tsx` (its 5 states, headings, fields, buttons, copy, confirm-first Deactivate, drop notice are all owned + tested in Phase 21).
- The pane wrapper must not double-pad: `LicenseSettings` already provides `flex flex-col gap-12 overflow-auto p-8`. Host it as the direct pane child so its own `overflow-auto` is the scroll container.
- Heading reconciliation (see Layout): the dialog header owns the visible `Settings` title; the selected nav item names the active pane (`License`); `LicenseSettings`'s `sr-only` `<h1>License</h1>` stays as the pane landmark. No visible "License" h1 is added.
- Unlicensed users: the License pane shows `LicenseSettings`'s `notActivated` state (Free heading + "Activate a license" CTA → `openUpsell()`), exactly as today — the modal being reachable by everyone (D-S10) means the no-license + Unlock-Pro state is reachable without a license.
