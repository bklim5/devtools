New `src/shell/useUpdater.ts` exports (shared singleton + hook):
```typescript
import { checkForUpdate, installUpdate } from "./update";
import { updatePreferences } from "./usePreferences"; // single-writer setter route
import type { UpdateInfo } from "@/lib/platform";

// Module singleton: updateInfo | null, status: string | null, checking: boolean,
//   installing: boolean, progress: number | undefined  + listeners/notify.

/** Run a check through the shared state machine. Stamps lastUpdateCheck on EVERY
 *  resolution (manual OR silent) so the pane readout is always accurate (D-25-6).
 *  `manual` controls whether a no-update/error surfaces a status string (manual
 *  gives feedback; the silent launch check stays quiet on current/error). */
export async function runUpdateCheck(manual: boolean): Promise<void> {
  setChecking(true);
  const result = await checkForUpdate();
  updatePreferences({ lastUpdateCheck: Date.now() }); // single-writer stamp, every check
  if (result.kind === "update") setUpdateInfo(result.info);
  else if (manual && result.kind === "current") setStatus("You're up to date");
  else if (manual && result.kind === "error") setStatus("Update check failed");
  setChecking(false);
}

export async function installPendingUpdate(): Promise<void> { /* lift handleInstall */ }
export function dismissUpdate(): void { /* setUpdateInfo(null) + reset progress */ }
export function setUpdateInfoForTest/* DEV inject */ ...

/** Hook: subscribes to the singleton, returns { updateInfo, status, checking,
 *  installing, progress, runCheck, install, dismiss }. */
export function useUpdater(): { ... };
```

Decision ‚Äî WHERE to stamp: stamp inside `runUpdateCheck` AFTER `checkForUpdate()` resolves (covers manual pane, tray, AND silent launch ‚Äî all three call this one function). Use `updatePreferences({ lastUpdateCheck: Date.now() })` directly (the single-writer module fn ‚Äî importable outside React, since `runUpdateCheck` is plain async, not a hook). This is the SAME singleton `setLastUpdateCheck` writes through, so no clobber.

App.tsx changes (behavior-IDENTICAL):
- Replace the local `updateInfo/installing/progress/status` useState + `runCheck`/`handleInstall` with `const { updateInfo, status, installing, progress, runCheck, install, dismiss } = useUpdater();`.
- The silent launch effect calls `runCheck(false)`; the tray listener calls `runCheck(true)`; the banner/opt-in/status JSX unchanged but driven by the hook values. The DEV `__injectUpdate` uses the hook's setter (or keep it pointing at the shared `setUpdateInfo` exported for the DEV seam).
- KEEP: `usePreferences()` for `preferences.autoUpdateCheck`/`prefsLoaded`/`setAutoUpdateCheck`, the opt-in prompt, the `onOpenSettings`/`useSettingsOpen`/`useUpsellOpen` wiring ‚Äî all unchanged.
