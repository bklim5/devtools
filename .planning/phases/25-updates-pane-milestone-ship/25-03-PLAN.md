The async-load clobber to AVOID (read `usePreferences.ts` lines 81-103 carefully):
`ensurePreferencesLoaded()` sets `dirty = true` on ANY `updatePreferences` call, and the deferred load does `if (!dirty) sharedPrefs = loaded`. So if the stamp writes `{ lastUpdateCheck }` via `updatePreferences` BEFORE the load resolves, `dirty` is already true → the real persisted blob (the user's theme/pins/license) is DISCARDED and only `DEFAULT_PREFERENCES + { lastUpdateCheck }` survives. This is the exact `tauri-store-async-init-race` + `prefs-blob-single-writer` clobber. The stamp MUST therefore wait for the real load to land first.

New `src/shell/usePreferences.ts` export (the load-safe primitive — added to the existing single-writer module):

```typescript
/** Resolves once the async mount-load of the persisted blob has landed (or
 *  immediately if it already has). Lets a non-React writer (the updater singleton's
 *  stamp) wait for the REAL persisted prefs before merging a field, so it can never
 *  persist DEFAULT_PREFERENCES+stamp over the user's real blob during the load
 *  window (memory tauri-store-async-init-race + prefs-blob-single-writer). It also
 *  KICKS the load (ensurePreferencesLoaded) so a stamp that fires before any hook
 *  mounted still resolves. */
export function whenPreferencesLoaded(): Promise<void>;
```

New `src/shell/useUpdater.ts` exports (shared singleton + hook):
```typescript
import { checkForUpdate, installUpdate } from "./update";
import { updatePreferences, whenPreferencesLoaded } from "./usePreferences"; // single-writer + load-gate
import type { UpdateInfo } from "@/lib/platform";

// Module singleton: updateInfo | null, status: string | null, checking: boolean,
//   installing: boolean, progress: number | undefined  + listeners/notify.
// Plus an in-flight guard: inFlight: Promise<void> | null.

/** Run a check through the shared state machine. DE-DUPES concurrent calls: if a
 *  check is already in flight, RETURN the same in-flight promise (do NOT call
 *  checkForUpdate again) so overlapping triggers (silent launch + tray + pane) hit
 *  platform.updater.check EXACTLY once and the final checking/status/updateInfo
 *  state is deterministic (D-25-3). Stamps lastUpdateCheck on EVERY resolution
 *  (manual OR silent) AFTER the real prefs load lands (load-safe — no defaults
 *  clobber). `manual` controls whether a no-update/error surfaces a status string. */
export function runUpdateCheck(manual: boolean): Promise<void> {
  if (inFlight) return inFlight;           // reuse the running check — one network call
  inFlight = (async () => {
    setChecking(true);
    try {
      const result = await checkForUpdate();
      await whenPreferencesLoaded();        // wait for the REAL blob before merging
      updatePreferences({ lastUpdateCheck: Date.now() }); // single-writer stamp, every check
      if (result.kind === "update") setUpdateInfo(result.info);
      else if (manual && result.kind === "current") setStatus("You're up to date");
      else if (manual && result.kind === "error") setStatus("Update check failed");
    } finally {
      setChecking(false);
      inFlight = null;                      // clear so the NEXT check can run
    }
  })();
  return inFlight;
}

export async function installPendingUpdate(): Promise<void> { /* lift handleInstall */ }
export function dismissUpdate(): void { /* setUpdateInfo(null) + reset progress */ }
export function setUpdateInfoForTest/* DEV inject */ ...
export function resetUpdaterForTest(): void { /* clear singleton + inFlight + listeners */ }

/** Hook: subscribes to the singleton, returns { updateInfo, status, checking,
 *  installing, progress, runCheck, install, dismiss }. */
export function useUpdater(): { ... };
```

Decision — IN-FLIGHT GUARD (Finding 1): the de-dupe is an explicit module-scoped `inFlight: Promise<void> | null`, NOT an assumption. The FIRST caller starts the check; any caller while it runs gets the SAME promise back and does NOT re-invoke `checkForUpdate()`. `manual` is captured by the first caller (a silent launch racing a manual tray click resolves to the first caller's manual flag — acceptable; both stamp, both share one network call). This is the de-dupe Plan 04 relies on (its T-25-13 "shared singleton de-dupes in-flight state").

Decision — WHERE/WHEN to stamp (Finding 2): stamp inside `runUpdateCheck` AFTER `checkForUpdate()` resolves AND AFTER `whenPreferencesLoaded()` resolves — for ALL three entry points (manual pane, tray, silent launch all call this one function). `whenPreferencesLoaded()` guarantees the user's real persisted blob has landed before `updatePreferences({ lastUpdateCheck })` merges the stamp, so the stamp can never persist `DEFAULT_PREFERENCES + timestamp` over the real prefs during the async-init race. Use `updatePreferences` (the single-writer module fn, importable outside React since `runUpdateCheck` is plain async) — the SAME singleton every setter writes through, so no second writer (prefs-blob-single-writer discipline holds).
