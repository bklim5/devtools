CE compose services (from official keygen-sh/keygen-api compose.yaml): `postgres` (postgres:17.5), `redis`, `web` (image `keygen/api:latest`, command `web`, internal port 3000), `worker` (image `keygen/api:latest`, command `worker`), `caddy` (ports 80/443, reverse_proxy web:3000, `tls internal` when CADDY_ACME_EMAIL unset), `setup` one-shot (command: `bundle exec rails keygen:setup`).

Required .env variables (names verbatim from keygen-api .env.sample):
```
POSTGRES_USER=keygen
POSTGRES_PASSWORD="$(openssl rand -hex 16)"
POSTGRES_DB=keygen
REDIS_URL=redis://redis:6379
SECRET_KEY_BASE="$(openssl rand -hex 64)"
ENCRYPTION_DETERMINISTIC_KEY="$(openssl rand -base64 32)"
ENCRYPTION_PRIMARY_KEY="$(openssl rand -base64 32)"
ENCRYPTION_KEY_DERIVATION_SALT="$(openssl rand -base64 32)"
KEYGEN_EDITION=CE
KEYGEN_MODE=singleplayer
KEYGEN_ACCOUNT_ID="$(uuidgen | tr 'A-Z' 'a-z')"   # MUST be lowercase
KEYGEN_ADMIN_EMAIL=admin@example.com
KEYGEN_ADMIN_PASSWORD="$(openssl rand -hex 16)"
KEYGEN_HOST=localhost                              # hostname, NOT an IP
CADDY_HOSTS=localhost
```

API endpoints (all under `https://localhost/v1/accounts/{KEYGEN_ACCOUNT_ID}` unless noted):
- Admin token: `POST https://localhost/v1/tokens` with HTTP Basic `email:password`
- Product: `POST /products` (admin Bearer token)
- Policy: `POST /policies` — payload MUST include `"authenticationStrategy": "LICENSE"` (defaults to TOKEN; without LICENSE every `Authorization: License <key>` call 403s), `"floating": false`, `"maxMachines": 1`, `"expirationStrategy": "RESTRICT_ACCESS"`
- License: `POST /licenses` (admin token; key autogenerates)
- Pubkey: `GET https://localhost/v1/accounts/{id}` (admin token) -> `data.meta.keys.ed25519` = BASE64 of the raw 32-byte public key (NOT hex)
- Validate: `POST /licenses/actions/validate-key` UNAUTHENTICATED, body `{"meta":{"key":"<key>","scope":{"fingerprint":"<hex>"}}}` -> `meta.valid`, `meta.code` (NO_MACHINE/NO_MACHINES = expected pre-activation, NOT failure)
- Activate: `POST /machines` header `Authorization: License <key>`, body `{"data":{"type":"machines","attributes":{"fingerprint":"<hex>","platform":"macOS","name":"Spike Mac"},"relationships":{"license":{"data":{"type":"licenses","id":"<license-id>"}}}}}`
- Token denial probe: `POST /licenses/{license-id}/tokens` header `Authorization: License <key>` -> EXPECT 403 (record full JSON)
- Checkout: `POST /machines/{machine-id}/actions/check-out?include=license,license.entitlements&ttl=2629746` header `Authorization: License <key>` -> `data.attributes.certificate` (the `-----BEGIN MACHINE FILE-----` text). Do NOT pass `encrypt=1`.
- Deactivate: `DELETE /machines/{machine-id}` header `Authorization: License <key>` -> 204

Caddy root cert extraction: `docker cp <caddy-container>:/data/caddy/pki/authorities/local/root.crt scripts/keygen-ce/caddy-root.crt` — all curl calls use `--cacert scripts/keygen-ce/caddy-root.crt` (or `-k` as last resort; prefer the cert so the same file feeds the Rust dev-trust path in Plan 03).

Known env hazards (Pitfall 8): Docker daemon was DOWN at research time; Docker CLI 20.10.7 + Compose 2.0.0-beta.3 (2021) — compose `profiles` unreliable on that beta. Avoid profiles entirely: name services explicitly in `up` commands and run `setup` via `docker compose run --rm setup`.
