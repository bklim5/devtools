Purpose: turn the app into a buildable, updater-emitting, ad-hoc-signed bundle whose Developer-ID activation is a future env-only flip. Almost all of DST-01/DST-02 is THIS configuration, not code.
Output: updated Cargo.toml/lib.rs/capabilities/tauri.conf.json + new entitlements.plist + a generated keypair (private gitignored, public committed).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Releases (remote) → updater | `latest.json` + `.app.tar.gz` are untrusted remote input fetched over HTTPS |
| build host → signed artifact | The minisign private key (env only) signs the updater payload |
| webview/tools → network | CSP governs what the webview may `connect-src`; must stay offline except the GitHub updater hosts |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering / Elevation | malicious/forged `.app.tar.gz` | mitigate | Committed `plugins.updater.pubkey` makes the plugin's mandatory minisign verify fail on any payload not signed by the matching private key (DST-02). Verify is non-disableable (V6 — not hand-rolled) |
| T-06-07 | Information disclosure | over-broad network egress | mitigate | CSP widened ONLY for `connect-src` and ONLY to `https://github.com` + `https://objects.githubusercontent.com`; no wildcard; tools never call them (Task 5, V9) |
