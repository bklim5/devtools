<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tray IPC event → JS shell | `menu://check-updates` is an app-internal event; it triggers a user-initiated check |
| update payload → app | The downloaded `.app.tar.gz` is untrusted; the plugin verifies the signature inside `downloadAndInstall` |
| user opt-in → network | Only an explicit opt-in (or a manual check) may cause a network call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-11 | Information disclosure | automatic network egress without consent | mitigate | Launch path calls `check()` ONLY when `shouldAutoCheck(pref) === true`; false/null → no call (Task 1 unit-asserted, Task 3 wiring). Offline-by-design preserved (D-09) |
| T-06-12 | Tampering / Elevation | forged update at install time | mitigate | `installUpdate` → `platform.updater.downloadAndInstall` runs the plugin's mandatory minisign verify before apply; a mismatch throws and surfaces an error state, never silently installs (DST-02) |
| T-06-13 | Denial of service | check()/install error crashing the shell | mitigate | `checkForUpdate` is error-as-value (never throws); install errors are caught and shown as a banner error state — the shell never crashes (Task 1/Task 3) |
| T-06-14 | Spoofing | seam leak via direct @tauri-apps import in the shell | mitigate | App.tsx/main.tsx/update.ts grep-asserted free of `@tauri-apps`; the tray-event listen + updater calls route through the platform seam (D-12) |
</threat_model>
