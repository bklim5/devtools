<decisions>
## Implementation Decisions

### Follow-up 1 вАФ app-menu product name (BUG)
- **D-22.1-1:** The macOS app menu must read **TinkerDev** everywhere: bold app-menu title + **About TinkerDev / Hide TinkerDev / Quit TinkerDev**.
- **D-22.1-2 (verified API):** In `set_menu()` (`src-tauri/src/lib.rs`), the App `SubmenuBuilder` title is ALREADY `"TinkerDev"` (line ~146). The wrong labels come from the *predefined items* `.about(None)` / `.hide()` / `.quit()`, whose no-text chainable forms derive their label from the executable name. **Fix:** build explicit `PredefinedMenuItem`s with custom text and `.item()` them instead of the no-text chainable forms. Verified against the installed crate `tauri-2.11.2/src/menu/predefined.rs`:
  - `PredefinedMenuItem::about(manager, text: Option<&str>, metadata: Option<AboutMetadata>)`
  - `PredefinedMenuItem::hide(manager, text: Option<&str>)`
  - `PredefinedMenuItem::quit(manager, text: Option<&str>)`
  Use `Some("About TinkerDev")` / `Some("Hide TinkerDev")` / `Some("Quit TinkerDev")`. Keep services/hide_others/show_all/separators as-is. Keep the existing `MenuMetadata`/about-metadata behavior (pass `None` for metadata unless trivially available).
- **D-22.1-3 (verify):** Native menu change вЖТ not WebDriver-drivable. Rebuild the `.app` and **manually re-verify** the menu labels at the human checkpoint. The Edit/Window submenus and the вМШ, Settings item must stay intact (no regression вАФ Pitfall 1).

### Follow-up 2 вАФ inline upsell/activation in the License pane (DESIGN)
- **D-22.1-4 (shared content, no duplicated logic):** Extract the upsell/activation surface from `UpsellPanel.tsx` so the **activation form** (license-key input + Activate + submitвЖТactivateвЖТclearEntitlementsOverrideвЖТrefreshLicenseUiвЖТrefreshEntitlements, stored-key affordance, inline error copy, the in-flight aria-live line, the "Licensed вАФ thank you" success state, the panel-mount `refreshLicenseUi`) lives in ONE reusable piece with NO logic duplicated. The standalone `UpsellModal` and the inline License pane both consume the shared surface. T-19-21 (key only in transient state, cleared on success) and LIC-04 (raw key never reaches JS) are preserved verbatim.
- **D-22.1-5 (standalone modal stays):** `UpsellModal` keeps its CURRENT behavior and entry points unchanged вАФ Sidebar "Unlock Pro" / locked-customization affordances and the вМШK free-tier "License" command still open the standalone modal at the shell. (ROADMAP SC3.) Only the **Settings вЦЄ License pane** no longer opens it stacked.
- **D-22.1-6 (free / notActivated вАФ full pitch inline):** In the License pane, the free/`notActivated` state renders the COMPLETE upsell surface inline вАФ "Thank you for using TinkerDev вЭ§пЄП" copy + **Buy license** CTA + the "I have a license key" reveal вЖТ key input + **Activate** вАФ identical content to what the standalone modal shows. No "Activate a license" button that opens a second modal.
- **D-22.1-7 (refreshNeeded / problem вАФ status+Refresh kept, form inline below):** For `refreshNeeded` ("Pro is no longer active") and `problem` ("License needs attention"), KEEP the existing calm status card copy + the **Refresh** button (these are distinct from activation), then render the **key-input + Activate** form INLINE beneath. Do NOT show the "Thank you вЭ§пЄП" sales pitch to a lapsed/attention paying customer (mirrors the D-44 principle). The current modal-opening **Reactivate** button is replaced by this inline form.
- **D-22.1-8 (Pro-active unchanged):** `licensed` / `offlineGrace` states are UNCHANGED вАФ status card + fields (masked key / email / renews via CopyButton) + the confirm-first inline **Deactivate**. No inline upsell for Pro states.
- **D-22.1-9 (a11y / focus):** WCAG-AA preserved вАФ real `<label>` for the key input, visible focus-visible rings, single `aria-live="polite"` status/error region, neutral tokens with accent reserved for the primary CTA + focus rings. Inline (non-modal) rendering means NO focus-trap is needed for the form in the License pane (the SettingsModal already owns the trap); focus must not be stolen/dropped. `problem` pre-reveals + focuses the field consistent with today's behavior. Class constants reused verbatim from `UpsellPanel`/`LicenseSettings` (no new sizes/tokens вАФ 22-UI-SPEC reuse mandate).
