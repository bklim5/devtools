# Phase 22.1 Plan 04: Standalone-Modal Removal + Dev E2E Seam Summary

Collapsed the licensing UI to a single upsell surface: the standalone Unlock-Pro
`UpsellModal` and its store/hook were deleted, and every former opener now routes
to the inline Settings ▸ License pane. Added a dev-only license-state injection
command so the real-WKWebView e2e can drive free/licensed/problem deterministically
without provisioning certs or touching the Keychain.

## What Shipped

- **Modal removal + opener redirect (`0e8c0d31`, `b947126a`):** removed the
  standalone `UpsellModal`; `upsellStore.ts` + `useUpsell.ts` deleted. The sidebar
  footer, every locked-customization affordance (pin click, Alt+↑/↓, Alt+P, reset),
  and the ⌘K free-tier "License" command now call `openSettings("license", invoker)`
  — the License pane renders the SAME inline `InlineActivation` the modal used to
  wrap. Tests inverted from `openUpsell` assertions to `openSettings` redirects;
  modal-specific tests dropped.
- **Dev e2e seam (`a18692a3`, `f8da92eb`):** `dev_set_license_state`
  (`cfg(debug_assertions)`, stripped from release; `check-dev-strip.sh` guards the
  prod literal absence) lets the e2e force free/licensed/problem. New
  `license-states.e2e.ts` + `setDevLicenseState` helper drive all three states on
  the real WKWebView.
- **Keychain heads-up (`0953a9da`):** a static muted line in the activation form
  warns that activation writes the key to the macOS Keychain and may raise a system
  prompt — pre-empts the surprise.
- **E2E repointing (`a770dd74`, `4b6ab755`):** modal-referencing specs repointed to
  the Settings ▸ License flow; confirm-card text probe, Alt+P→Settings wait,
  override re-sync, and entitlements re-resolve fixed for the locked-Alt+P proof.
