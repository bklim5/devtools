# Phase 22.1 Plan 01: App-Menu Product Name Summary

Closed Follow-up 1 (BUG): the macOS application menu's About / Hide / Quit items
read the Cargo binary name "devtools-app" instead of the product name. The phase-22
menu rebuild used the no-text chainable `PredefinedMenuItem` forms
(`.about(None)`, `.hide()`, `.quit()`), which fall back to the bin name; this plan
builds them explicitly with the product-name text.

## What Shipped

- **Task 1 — explicit product-name menu items (`d9dba1f1`):** `src-tauri/src/lib.rs`
  imports `PredefinedMenuItem` and constructs
  `PredefinedMenuItem::about(app, Some("About TinkerDev"), None)`,
  `::hide(app, Some("Hide TinkerDev"))`, `::quit(app, Some("Quit TinkerDev"))`, and
  wires them into the first (macOS application) submenu. The App submenu now reads
  "About TinkerDev / Hide TinkerDev / Quit TinkerDev". Edit/Window submenus and the
  ⌘, Settings item are byte-unchanged (the full-rebuild regression guard holds).

## Verification

- **Unit/typecheck:** `cargo` builds clean; webview suite + `tsc --noEmit` unaffected
  (Rust-only change). Decoder + its 19 tests untouched.
- **Human walkthrough:** APPROVED — native menu labels cannot be asserted by WebDriver
  (the human walkthrough is the backstop per the file header); the user confirmed the
  App submenu reads "TinkerDev" on the built app.
