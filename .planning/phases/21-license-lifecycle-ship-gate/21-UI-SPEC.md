# Phase 21 — UI Design Contract

> Visual and interaction contract for the License Lifecycle & Ship Gate phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase ships ONE new app-chrome surface — a **license status / settings route** (D-87, outside the six-tools/registry constraint, HashRouter-friendly) — plus state-driven changes to existing chrome (footer license-attention affordance D-88, command-palette routing, one-time drop notice D-84, the calm "Pro is no longer active" state D-83). It does **not** invent a new visual system: every token, class, and interaction pattern below derives from the canonical `design/DevTools Mockup.html` (mapped into `src/index.css` `@theme`) and from the already-shipped Phase 18/19 surfaces (`UpsellPanel`, `UpsellModal`, the D-29 footer row). Reuse over reinvention is mandatory.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — Tauri 2 + Vite + React; **zero new webview runtime/dev deps** is a binding constraint) |
| Preset | not applicable |
| Component library | none — hand-rolled components styled with Tailwind v4 utilities; reuses in-repo `UpsellPanel`/`UpsellModal`, `CopyButton`, `SegmentedControl` |
| Styling | Tailwind v4 **CSS-first** (`@import "tailwindcss"` + `@theme` in `src/index.css`; **NO `tailwind.config.js`, NO `postcss.config`**). Tokens generate utilities like `bg-panel`, `text-tx-2`, `border-bd`, `text-accent`, `bg-accent-soft`, `ring-accent` |
| Icon library | `lucide-react` (already a dep; `Lock` glyph established for license/Pro surfaces; neutral `tx-2` color, accent never used for status glyphs) |
| Font | Self-hosted **IBM Plex Sans** (body/UI) + **JetBrains Mono** (keys, code, byte values), vendored via `@fontsource` static weights — **zero runtime network / no CDN** (the mockup's Google Fonts CDN is deliberately NOT carried over) |
| Routing | **HashRouter only** (`createHashRouter`). The new status route is registered as **app chrome** — NOT derived from `ENABLED_TOOLS` (it is not a tool; the registry stays the single control plane for the six/eleven tools). It lives as a non-tool child route under `App`, e.g. `#/settings/license` |
| Theme | Dark only. The status route renders on `--color-pane` (`#14161b`) like the main tool area; cards on `--color-panel`/`--color-card` (`#181b21`) |

## Spacing Scale

Declared values (multiples of 4; matches the mockup's `--gap: 16px`, card `--field-pad`, and the existing `UpsellPanel` `gap-4`/`p-6` rhythm):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing, button row gaps (`gap-2`) |
| md | 16px | Default element spacing, gap between status-card rows (`gap-4`) |
| lg | 24px | Card inner padding / section padding (`p-6` = 24px on the status cards, matching `UpsellPanel`) |
| xl | 32px | Route content outer padding (`--pad` family; align with `.toolhead` 28–32px) |
| 2xl | 48px | Major vertical break between status block and management actions |
| 3xl | 64px | Reserved — not expected on this single-column route |

Exceptions:
- **Touch / focus targets ≥ 24×24px** for icon-affordance parity (the established WCAG 2.5.8 minimum used for sidebar pin/grip and the footer row's `min-h-6`). The footer license-attention button keeps `min-h-6` (24px).
- Status-route action buttons use the existing `px-3 py-1` (12px / 4px) button padding from `UpsellPanel` `PRIMARY_BTN_CLASS`/`SECONDARY_BTN_CLASS` — do not introduce a new button size.

## Typography

Three sizes, two weights (mirrors the shipped `UpsellPanel`: 16px semibold heading + 12px regular body; mono for key/email/code). Line-heights: body 1.5, heading 1.2 — exactly the values already in `HEADING_CLASS`/`BODY_CLASS`.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Heading | 16px | 600 (semibold) | 1.2 | Route title ("License") and card headings — reuse `text-[16px] font-semibold leading-[1.2] text-tx` (UpsellPanel `HEADING_CLASS`) |
| Body | 12px | 400 (regular) | 1.5 | Status descriptions, helper lines, button labels — reuse `text-[12px] leading-[1.5] text-tx-2` (UpsellPanel `BODY_CLASS`); buttons use `text-[12px]` |
| Mono (data) | 12px | 400 (regular) | 1.5 | Masked key + licensee email + expiry/refresh dates — `font-mono text-[12px]` (matches the key `<input>` and field-value treatment); the masked key is JetBrains Mono so the last-N chars are unambiguous |

Field labels ("License key", "Licensee", "Status") reuse the existing `text-[12px] text-tx-2` label treatment (same as the activation form's `<label>`). Do not add a fourth size or a third weight; the 11px uppercase `pane-label` tracking is available only if a section eyebrow is needed (reuse `.pane-label` semantics, `text-[11px] uppercase tracking-[.09em] text-tx-3 font-semibold`) and counts as a label variant, not a new role.

## Color

60/30/10 split inherited verbatim from `src/index.css` `@theme`. Accent stays **selection/primary-action only** — this is an app-wide discipline (D-24): status glyphs and state indicators are NEVER accent.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-pane` `#14161b` (route surface); `--color-bg-app` `#0a0b0d` (window) | Route background, window |
| Secondary (30%) | `--color-panel` / `--color-card` `#181b21`; `--color-input-bg` `#0d0f13` | Status cards, the route's content cards, the masked-key/email field surfaces, footer |
| Accent (10%) | `--color-accent` `#5b9bf8` (+ `--color-accent-soft` 15% fill, `--color-accent-line` 75% border, `--color-ring` accent) | See reserved list below |
| Semantic — OK | `--color-ok` `#34d399` | The "Licensed / active" status dot ONLY (matches the StatusBar `.dot`/`.st-ok` precedent). Calm, not celebratory |
| Semantic — Attention | `--color-bad` `#f0876b` | Reserved for the **inline activation/error line** (the established `text-bad` `role=alert` tint) and the destructive-confirm copy. **NOT** used for OfflineGrace / RefreshNeeded / revoked states — those are CALM neutral `tx-2`, never alarm-colored (D-77/D-83/D-84; ENT-04 no alarm styling). No red borders, no banners. |

Accent reserved for (explicit — never "all interactive elements"):
1. The **primary action button** fill/border on the status route — i.e. the `Refresh` action and the reactivation/"Activate" path reuse `PRIMARY_BTN_CLASS` (`border-accent-line bg-accent-soft text-accent`).
2. **`focus-visible` rings** on every interactive control (`focus-visible:ring-2 focus-visible:ring-accent`) — keyboard-reachability is mandatory, WCAG-AA.
3. The **navbar selected-state** accent bar / active label if the status route appears as a selected nav entry (same `navitem.on` accent treatment the tools use).

Everything else — status glyphs (`Lock`), the masked key, the licensee email, state descriptions, the `Deactivate` secondary button, the offline-grace / refresh-needed / "no longer active" indicators, the one-time drop notice — is **neutral** (`tx`, `tx-2`, `tx-3`, `bd`, `bd-2`). Calm tone is binding (D-34/37/43): **no toasts, no spinners, no launch interruptions, no opacity-only locked state** (ENT-04). In-flight feedback is a calm `aria-live="polite"` text line (e.g. "Refreshing…"), never spinner chrome — exactly the D-34 pattern already in `UpsellPanel`.

## Copywriting Contract

Calm, non-accusatory, device-neutral ("device", never "Mac" — locked by the 2026-06-12 walkthrough decision so copy survives the cross-platform future). All copy below is the contract; the executor uses these verbatim unless a checker/walkthrough revision is recorded.

### Status route — state copy (one row each, reads from verified cert data)

| State (D-73) | Status label | Body line |
|--------------|--------------|-----------|
| `notActivated` (free) | "Free" | "Most of TinkerDev is free. Activate a license to unlock custom themes and tool reordering." (route shows the activation path / routes to the Unlock Pro panel — D-88) |
| `licensed` | "Licensed" (with the OK dot) | "Pro is active on this device." + a `tx-3` line: "Licensed to {email} · key {masked} · refreshes automatically." |
| `offlineGrace` (D-73/D-77) | "Licensed (offline)" | "Pro is active. We'll refresh your license automatically the next time you're online." — CALM, neutral; no countdown alarm. |
| `refreshNeeded` (D-73/D-83) | "Pro is no longer active" | "Connect to the internet and refresh to restore Pro. Your themes and tool order are saved and will come right back." (ONE calm state regardless of cause — revocation vs expiry — D-83; never exposes refund/revocation reason) |
| `problem` (corrupt/foreign lic, D-44) | "License needs attention" | "Your license file couldn't be verified. Your tools keep working — activate again to restore your license." (reuses the existing D-44 UpsellPanel copy) |

### Actions, fields, and notices

| Element | Copy |
|---------|------|
| Primary CTA — Refresh | **"Refresh"** (button). In-flight `aria-live` line: "Refreshing…". On a refresh that drops entitlements: silently transitions to the `refreshNeeded`/"Pro is no longer active" state — no error dialog (D-82). |
| Primary CTA — Reactivate (refreshNeeded / problem) | **"Reactivate"** → routes to / reveals the activation form (reuses the D-33 key field). Same action regardless of cause (D-83). |
| Field — Licensee | Label "Licensee"; value = the licensee email from verified cert data (D-89), `font-mono text-[12px]`. If absent (older license pre-D-89): show "—" (em dash), never empty. |
| Field — Key | Label "License key"; value = the **masked key** (last-N chars, Rust-side; format is Claude's discretion per D-89, recommend `••••••••{last4}`). The raw key NEVER round-trips through JS (LIC-04). |
| Field — Expiry/refresh | Label "Renews"; value = a human date derived from `expiry` (e.g. "Renews around {date}") in `tx-3`. Omit the row entirely for `notActivated`/`free`. |
| Empty state heading | (route never truly "empty" — `notActivated` IS the free state) "Free" |
| Empty state body | "Most of TinkerDev is free. Activate a license to unlock custom themes and tool reordering." |
| Error state | Inline `aria-live` line under the relevant control, `text-bad` (D-37). Reuse the existing `ERROR_COPY` map verbatim for activation/refresh: offline = "You're offline — connect and try again."; serviceUnreachable = "Can't reach the licensing service — try again shortly."; seatLimit (extended for D-80, see below). |
| One-time drop notice (D-84) | A calm, **dismissable** inline notice on next open when entitlements drop (NOT a toast/dialog mid-use). Copy: "Your Pro features turned off. Your themes and tool order are saved — reactivate any time to bring them back." Neutral tokens; a single "Dismiss" / "Got it" affordance (reuse `SECONDARY_BTN_CLASS`). Dismiss mechanism is Claude's discretion within the calm-tone constraint (recommend a one-shot prefs flag, surfaced inline on the status route and/or a quiet footer reflection — never a launch interruption D-77). |

### Destructive action

| Action | Confirmation copy |
|--------|-------------------|
| **Deactivate** (D-78, confirm-first inline) | Trigger button **"Deactivate this device"** (`SECONDARY_BTN_CLASS`, neutral — NOT a red destructive button; calm). Clicking reveals a **calm inline confirm** in place (no separate modal/route — mirrors the D-33 reveal-in-place pattern), NOT a browser confirm. Confirm copy: "This frees your seat so you can activate another device. Pro turns off here until you reactivate." Confirm button **"Deactivate"** + Cancel **"Keep Pro here"**. On confirm → server delete-machine → clear Keychain + `machine.lic` → drop to free; in-flight `aria-live` line "Deactivating…". |
| **Deactivate while offline** (D-79) | The confirm/Deactivate path is **blocked** with a calm message (local state is NEVER cleared offline): "Connect to the internet to free this seat." Render inline in the same `aria-live` region, `text-tx-2` (calm, not `text-bad` — it's guidance, not an error). |
| **Seat-taken on a new device** (D-80, extends `ERROR_COPY.seatLimit`) | Two calm lines, inline below the key field: (1) "This key is active on another device. There: open DevTools → License → Deactivate, then activate here." (2) "Lost access to that device? Reply to your license email and we'll free the seat." |

## Interaction & Layout Contract (phase-specific)

This phase is interaction-heavy; these are binding contracts the executor implements and the checker/auditor verify.

- **New route is app chrome, not a tool.** Registered as a non-tool child route (e.g. `#/settings/license`), reached from (a) the footer license-attention / Unlock-Pro affordance and (b) the command palette — routing target depends on state (D-88): `licensed`/`offlineGrace`/`refreshNeeded` → **status route**; `notActivated` (free) → the **Unlock Pro panel** (activation pitch). The status route owns post-activation status + management; the panel owns the sales pitch + the D-33 activation form (D-88).
- **Single-column, scrollable, max-width content** consistent with the tool panes: a status block (state label + OK dot when licensed + the licensee/key/renews fields) above a management block (`Refresh`, `Deactivate this device`, and `Reactivate` when applicable), separated by 48px (2xl) vertical space. Cards reuse `CARD_CLASS` (`rounded-[7px] border border-bd bg-panel p-6`) and the `max-w-[420px]` measure where a card is used.
- **Keyboard-reachable + WCAG-AA (mandatory, not optional):** every control is a native `<button>`/`<input>` Tab stop with a visible `focus-visible:ring-2 focus-visible:ring-accent`; the route is reachable by keyboard from the footer affordance and the palette; the inline Deactivate-confirm moves focus to its confirm control on reveal and returns focus on cancel/complete (mirror the `UpsellModal` focus-capture/return contract). `aria-live="polite"` carries every in-flight status and result line (no spinners).
- **Calm state changes:** entitlement flips (activate / refresh-success / refresh-drop / deactivate) reuse the proven `refreshEntitlements()` + `refreshLicenseUi()` live-flip path — **no restart, no toast, no launch interruption** (D-76/D-77/D-82). OfflineGrace is **silent outside this route** (no footer nag — D-77); only `problem` and `refreshNeeded` surface through the footer license-attention affordance (D-84).
- **Footer affordance (existing, extended):** the Sidebar D-29/D-43 row keeps its neutral `Lock` glyph + `min-h-6`, `text-tx-2 hover:text-tx`, accent only on focus ring. Its label reflects state: "Unlock Pro" (free), "License needs attention" (`problem`/`refreshNeeded`). It routes per D-88. No new alarm styling.
- **Locked features stay visible** (ENT-04) when the free-tier flip goes live (D-85): theming/ordering/pinning affordances render with the existing neutral lock treatment and open the upsell — never hidden, no opacity-only state. Customizations are preserved dormant (D-86): locked → default render, stored prefs untouched, instant restore on activate.
