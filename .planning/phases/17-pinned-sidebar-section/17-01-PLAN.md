<objective>
Add the persistence + pure pinning/partition backbone for the Pinned sidebar section —
with NO UI yet. This is the contract layer the Sidebar (Plan 17-02) consumes.

Four pieces:
1. A `pinnedToolIds: string[]` field on `Preferences` (default `[]`), persisted through
   the existing prefs blob exactly like `toolOrder` — including untrusted-merge coercion
   (`coercePinnedToolIds`) in `prefsStore.ts`.
2. `setPinnedToolIds` + `togglePinned` setters in `usePreferences.ts`, mirroring `setToolOrder`.
   `togglePinned` appends-on-pin (bottom of the pinned group) and removes-on-unpin.
3. A pure, unit-tested `partitionTools(pinnedToolIds, toolOrder, registryIds)` in
   `src/shell/toolOrder.ts` returning `{ pinned, unpinned }` — the REORD-style immovable
   bar: output is ALWAYS a full registry partition (every tool in exactly one group),
   reusing `reconcileToolOrder` for the unpinned remainder.
4. Full vitest coverage: the `partitionTools` immovable-bar matrix, `coercePinnedToolIds`
   cases, and the `setPinnedToolIds`/`togglePinned` round-trip.

Purpose: keep the WCAG-AA keyboard path and the PIN-08 reconciliation as a pure function
with real test coverage, isolated from React/DOM — so Plan 17-02's Sidebar is thin wiring.
Output: `pinnedToolIds` field + coercion + setters + `partitionTools` + their tests.

Requirements: PIN-07 (persisted pinnedToolIds via the existing seam + safe coercion),
PIN-08 (unknown/removed IDs dropped, duplicates collapsed, full registry permutation —
no crash, no missing or duplicated tool). Pure backbone enabling PIN-01/02/06.
