# Phase 22.2: Pro-gate ⌘K + focused upsell modal - Context

**Gathered:** 2026-06-16
**Status:** Ready for planning

<domain>
## Phase Boundary

Two coupled UX/gating changes the user approved on 2026-06-16, reconsidering part of
Phase 22.1:

1. **Gate the ⌘K command palette behind Pro.** A free user who presses ⌘K (or clicks
   the header ⌘K pill) gets a **focused "Unlock Pro" modal** instead of the palette. A
   Pro user gets the palette exactly as today.
2. **Route the CONTEXTUAL locked customization triggers** (pin click, drag-reorder,
   Alt+P, "Reset order") to that **same focused modal** — instead of the full
   Settings ▸ License redirect that 22.1 shipped for them.

The focused modal reuses the existing shared `ActivationSurface` (no duplicated
activation logic) behind a thin dialog wrapper + a minimal shared trigger store. The
EXPLICIT license entry points stay as-is (free → Settings ▸ License) so a free user
can still reach the buy/activate flow.

NOT in scope: making the whole product Pro, tiered licensing, any Appearance/theme
work (that is Phase 23), Keygen policy/cert changes.
</domain>

<decisions>
## Implementation Decisions

### ⌘K gating
- **D-22.2-1:** The ⌘K command palette is **Pro-only**. Free users cannot open it.
  This revises SET-04 ("⌘K opens for everyone, incl. unlicensed") and reframes ⌘K as a
  Pro "power navigation" feature alongside theming + ordering. (User accepted the
  positioning shift knowingly: ⌘K is the primary fast tool-switcher; free users switch
  tools via the sidebar — click or ↑/↓ arrows — which is unchanged.)
- **D-22.2-2:** Gate via a NEW predicate `isPro(set: EntitlementSet): boolean` in
  `src/lib/entitlements/entitlements.ts`, defined as "has any Pro entitlement"
  (`set.size > 0`, equivalently `set.has(ENT_THEMING) || set.has(ENT_ORDERING)`). The
  $9 license grants theming+ordering together, so "has any" == "is a paying customer".
  **Frontend-only — do NOT add a new cert entitlement code** (`resolve.ts:30` intersects
  the cert's `entitlements` with `ALL_ENTITLEMENTS`, so a new code would lock ⌘K for
  EXISTING customers until their `machine.lic` is re-issued/refreshed — a regression +
  Keygen churn we explicitly avoid). The DEV override (`"free"`→FREE_SET, `"full"`→
  FULL_SET) still drives both states for e2e.
- **D-22.2-3:** The gate lives at the single ⌘K open trigger — the global keydown
  handler in `CommandPalette.tsx` (`~:171`, `(e.metaKey||e.ctrlKey) && key==='k'`). The
  header pill (`App.tsx:35 openPalette`) dispatches the same synthetic ⌘K event, so one
  gate covers both. If `!isPro` → open the focused Unlock-Pro modal instead of the
  palette. (Consider a lock affordance on the header pill for free users — Claude's
  discretion / UI polish.)

### Focused Unlock-Pro modal
- **D-22.2-4:** Re-introduce a thin **modal wrapper** around the existing shared
  `ActivationSurface` "panel" variant — resurrect/adapt the standalone `UpsellModal`
  removed in commit `0e8c0d31` (git history is the reference). It MUST provide: focus
  trap, Esc-to-close, return-focus-to-invoker, a scrim/`aria-modal` dialog. It renders
  the SAME `ActivationSurface` the Settings ▸ License pane uses — **no duplicated
  activation logic** (activate-chain grep stays == 1). "One activation surface, two
  presentations" — this only PARTIALLY un-reverts D-22.1-5 (the *logic* stays single).
- **D-22.2-5:** The modal must NOT stack on top of the Settings modal. The contextual
  triggers + ⌘K fire from the main UI (Settings closed), so this is structural; guard
  it the way `stackedUpsellModalPresent()` asserts in e2e.
- **D-22.2-6:** ONE minimal shared trigger — a small `upsellStore` (open/close + the
  invoker element for focus-return), mirroring the `settingsStore` useSyncExternalStore
  pattern. BOTH the ⌘K handler AND the Sidebar contextual triggers call it. The modal is
  mounted once at the shell level (like the Settings modal / CommandPalette). This is
  why sidebar-local state is insufficient (the ⌘K trigger is not in the Sidebar — the
  user's original instinct).

### Contextual locked triggers
- **D-22.2-7:** `Sidebar.tsx` pin click, drag-reorder (`useSidebarDragDrop`), Alt+P,
  and "Reset order" — currently `openSettings("license", invoker)` via
  `openOrderingUpsell` (22.1) — now open the focused modal via the shared store,
  threading the SAME explicit invoker for focus-return (keep the 22.1 MED-22-02 work).

### Explicit license entry points (UNCHANGED — stay free → Settings ▸ License)
- **D-22.2-8:** The sidebar "Settings" row, the sidebar "Unlock Pro" footer, the
  app-menu ⌘, / tray "Settings", and the `#/settings/license` deep link continue to open
  Settings ▸ License and remain FREE-reachable, so a free user always has a path to buy.
  The focused modal also offers Buy + "I have a license key" → activate inline.

### Copy
- **D-22.2-9:** Add the command palette as a Pro feature in the upsell pitch
  (`UpsellPanel` `PITCH_FEATURES`) — e.g. "Command palette — jump to any tool, no
  mouse". Keep the pitch design coherent (it currently lists 3 features → 4). **Flag the
  tinkerdev.io Pro card for a matching copy update** (separate repo `../tinkerdev-io`,
  not part of this phase's git history — mirror the D-20 pricing-sync note).

### Claude's Discretion
- Exact `upsellStore` shape + whether to fully resurrect `useUpsell.ts` or inline a
  smaller hook.
- Whether the header ⌘K pill shows a lock for free users.
- Exact pitch feature wording/icon for the command-palette row.
- e2e structure (a new `cmdk-pro.e2e.ts` vs. extending existing specs); reuse the
  `dev_set_license_state` seam + the DEV free/full override to drive both states.
</decisions>

- **Git history `UpsellModal` (pre-`0e8c0d31`)** — the focus-trap/Esc/return-focus dialog wrapper to resurrect.
- **22.1 focus-return invoker threading** — `togglePin(id, invokerEl)`, `getRowEl`, locked-Alt `e.currentTarget` — reused, just pointed at the modal store instead of `openSettings`.
- **`isToolLocked`/`isEntitled`/`gatePreferences`** — entitlement predicates; add `isPro` beside them.

### Established Patterns
- Entitlements resolve from the cert's `entitlements` ∩ `ALL_ENTITLEMENTS` (`resolve.ts`) — the reason `isPro` (no new code) is the correct mechanism.
- DEV override `"free"`/`"full"` drives FREE/FULL for e2e; `dev_set_license_state` seam drives license states.
- One shell-mounted overlay per surface (Settings modal, CommandPalette) — the focused upsell modal joins them.

### Integration Points
- ⌘K keydown handler (`CommandPalette.tsx`) + header pill (`App.tsx`) → gate → `upsellStore.open(invoker)`.
- Sidebar `openOrderingUpsell` + drag locked path → `upsellStore.open(invoker)` (was `openSettings`).
- Shell root (`App.tsx`) mounts the `<UpsellModal/>` reading `upsellStore`.
</code_context>

<specifics>
## Specific Ideas

- "One activation surface, two presentations" — the modal and the Settings ▸ License pane must render the IDENTICAL `ActivationSurface`; the only difference is the wrapper chrome.
- Contextual triggers should feel like a quick interruption that returns you exactly where you were (Esc → back to the row/control), NOT a navigation into Settings.
</specifics>
