# Phase 22.2: Pro-gate ⌘K + focused Unlock-Pro modal Summary

User-approved mid-Phase-23 scope change (reconsidering the 22.1 "redirect every
locked trigger to Settings" decision): the ⌘K command palette is now a Pro feature,
and the contextual locked customization triggers open a focused Unlock-Pro modal
instead of yanking the user into full Settings. One shared activation surface, two
presentations.

## What Shipped

- **isPro gate (`entitlements.ts`):** `isPro(set) = set.size > 0` — "any Pro
  entitlement == Pro" (the $9 license grants the full set together). Frontend-only:
  no new cert code, so every existing license keeps ⌘K working immediately. Constraint
  (one-tier-all-features) documented.
- **⌘K gate (`CommandPalette.tsx`):** the keydown handler routes a not-Pro user to
  `openProUpsell` instead of opening the palette; a DEV-only `⌘⇧K` force-open escape
  keeps the in-palette dev toggle reachable while free (tree-shaken from release). The
  header ⌘K pill dispatches the same synthetic ⌘K, so one gate covers both.
- **Focused modal (`UpsellModal`, restored):** a thin focus-trapped/Esc/return-focus
  dialog over the shared `ActivationSurface` (panel variant) — no duplicated activation
  logic. Mounted once at the shell (`App.tsx`) via the restored `upsellStore`/`useUpsell`.
- **State-aware routing (`proUpsell.ts`):** `openProUpsell` sends genuinely-free
  (notActivated) users to the focused pitch modal, and lapsed/attention PAYING customers
  (refreshNeeded/problem) to the Settings ▸ License recovery form — never the sales
  pitch (D-44). Used by both the ⌘K gate and the Sidebar contextual triggers.
- **Sidebar contextual triggers:** pin/drag/Alt+P/Reset → `openProUpsell` (was
  `openSettings` in 22.1); the explicit footer + Settings row stay → `openSettings`.
- **Pitch copy:** added a "Command palette" Pro feature row to `PITCH_FEATURES`.
