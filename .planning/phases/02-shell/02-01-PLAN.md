<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted prefs ↔ app | The on-disk `prefs.json` (or localStorage) is user-editable; values read back are untrusted input |
| Tauri JS ↔ Rust plugin | Plugin commands are gated by capabilities; an un-granted command rejects at runtime |
| npm/crates supply chain ↔ build | Two new third-party deps enter the build graph |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | src-tauri/capabilities/default.json | mitigate | Grant only `store:default` (scoped to store commands); no broader/unrelated permissions added (Task 1) |
| T-02-02 | Tampering | platform store get() | mitigate | Treat stored values as untrusted: corrupt/non-JSON values degrade to `undefined` rather than throwing or being trusted (Task 2 behavior + test) |
| T-02-03 | Tampering | supply chain (lucide-react, plugin-store) | mitigate | Pin exact versions (`1.17.0`, `2.4.3`); both are mainstream official/widely-used packages; pnpm lockfile records integrity hashes (Task 1) |
| T-02-04 | Denial of Service | disk writes on the hot path | accept | autoSave debounces writes; per-switch (not per-render) writes land in Plan 03 — low risk for a single-user local app (RESEARCH Pitfall 5 / Assumption A3) |
</threat_model>
