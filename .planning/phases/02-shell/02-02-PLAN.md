<feature>
  <name>rankTools — subsequence fuzzy ranker</name>
  <files>src/shell/fuzzy.ts, src/shell/fuzzy.test.ts</files>
  <behavior>
    Signature: `rankTools(query: string, tools: ToolDefinition[]): ToolDefinition[]`.
    - Empty/whitespace query → return `tools` unchanged (the caller layers recents/registry order on top; D-05).
    - Non-empty query, lower-cased: for each tool, attempt to match the query as a case-insensitive SUBSEQUENCE against the tool's searchable fields, with a field weight ordering name > keywords > description (a match in `name` outscores the same match in `description`).
    - Score rewards: contiguous runs of matched chars, earlier match position, and a match that begins at a word boundary / field start. (Exact scoring formula is Claude's discretion — D-06 — but the ORDERING guarantees below are the contract the tests assert.)
    - Tools with NO subsequence match in any field are excluded from the result (so an unknown query → empty array → powers the palette's quiet "No tools match", D-07).
    - Result is sorted by descending score; ties broken by registry order (stable).
    Test cases (fuzzy.test.ts), using small inline fixture ToolDefinitions (do NOT depend on the live registry):
    - `rankTools("", tools)` returns all tools in input order.
    - `rankTools("b64", [base64-ish, other])` matches the base64 tool (subsequence b→6→4 across name/keywords) and excludes a non-matching tool.
    - A query matching a tool's NAME ranks it above a tool where the same query only matches its DESCRIPTION.
    - A contiguous match (e.g. "proto") ranks above a scattered subsequence match of the same query length.
    - `rankTools("zzzzz", tools)` returns `[]`.
    - Matching is case-insensitive ("PROTO" === "proto").
  </behavior>
  <implementation>
    Pure module, no React, no platform/store, no @tauri-apps. Import only the `ToolDefinition` type. Build a per-field lowercased haystack; implement a small `subsequenceScore(needle, haystack)` returning a number or null (null = no match), then combine field scores with the name>keywords>description weighting. Keep it ~30-50 lines. Export `rankTools` (and optionally the internal `subsequenceScore` for unit coverage). RED: write fuzzy.test.ts asserting the contract above and watch it fail. GREEN: implement rankTools until green. REFACTOR: tidy scoring if needed, tests stay green.
  </implementation>
</feature>
