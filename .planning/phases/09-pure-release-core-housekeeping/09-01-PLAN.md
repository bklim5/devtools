    `bumpSemver(version, level)` (D-02/D-02a): parse `MAJOR.MINOR.PATCH` via a strict regex like `/^(\d+)\.(\d+)\.(\d+)$/` (NO prerelease/build-metadata). On no match, throw `Error("Invalid semver: <version>")`. Switch on level: `patch` -> `M.m.(p+1)`; `minor` -> `M.(m+1).0`; `major` -> `(M+1).0.0`. On unknown level throw `Error("Unknown bump level: <level>")`. This is the single computed source of truth for the next version.

    The three editors are SURGICAL string->string replacements (D-01) that rewrite ONLY the version line and preserve all surrounding formatting/key order/indentation:
    - `setPackageJsonVersion(content, newVersion)` and `setTauriConfVersion(content, newVersion)`: match the FIRST top-level `"version"` key (D-01a). Anchor to a key at the top-level indentation (e.g. a 2-space-indented `"version": "..."`), e.g. regex `/^(\s*"version"\s*:\s*")[^"]*(")/m`. Count matches; if 0 or >1 at the targeted scope, throw a clear Error (`"package.json: expected exactly one top-level \"version\", found N"`). Replace only the captured value.
    - `setCargoVersion(content, newVersion)`: match the `version` line under the `[package]` section ONLY (D-01a) — NEVER a dependency-section `version = "..."`. Locate the `[package]` header, then within that section (up to the next `[` section header) match `/^version\s*=\s*"[^"]*"/m` and replace the value. Dependency pins like `serde_json = "1"`, `tauri = { version = "2", ... }`, `tauri-plugin-store = "2.4.3"` MUST remain byte-identical. Throw on 0 or >1 `[package]` version matches.

    Every throw path must produce a clear, file-named message so a malformed/unexpected manifest fails loudly rather than silently no-op'ing (T-09-01).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest-file -> version editors | The project's OWN manifest text (package.json / tauri.conf.json / Cargo.toml) is read and rewritten. Not untrusted user input at runtime, but a malformed/unexpected file shape could cause a silent wrong edit. |
| CI/caller string -> bumpSemver | The bump level + base version are supplied by the Phase 10 driver / maintainer, not an end user. Malformed values must fail loudly, not silently corrupt a version. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | setCargoVersion / setPackageJsonVersion / setTauriConfVersion | mitigate | Anchor precisely (first top-level "version"; `[package]`-scoped Cargo version) and THROW on 0 or >1 matches (D-01a) so a malformed/unexpected manifest fails loudly instead of silently editing the wrong line or no-op'ing. Unit-tested incl. the dependency-pin-untouched proof. |
| T-09-02 | Tampering | bumpSemver | mitigate | Strict `^\d+\.\d+\.\d+$` parse + throw on malformed input / unknown level (D-02a). No prerelease/build-metadata surface to mis-handle (YAGNI). |
| T-09-03 | Denial of Service (corruption) | Cargo reconcile (D-07) | accept | One-time local dev edit against the repo's own file, immediately diff-verified (only line 3 changes) and covered by version control (revertible). No runtime/network exposure. |
| T-09-04 | Information Disclosure | latest.json untrack (D-06) | accept | latest.json contains only a public signature + public download URL (already public on the releases repo); untracking is a hygiene step, not a secret-leak fix. No private key is ever in scope here. |
</threat_model>
