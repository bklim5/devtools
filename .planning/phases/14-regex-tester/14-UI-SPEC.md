# Phase 14 ‚Äî UI Design Contract

> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

The shared shell already uses a 4-based scale via Tailwind utilities. This tool reuses it; the
sibling `UrlTool.tsx` is the precedent (`gap-4`, `p-4`, `p-3`, `gap-2`, `gap-1.5`, `gap-3`).

| Token | Value | Usage in this tool |
|-------|-------|--------------------|
| xs | 4px | `gap-1` between flag toggle buttons / chip-row items; capture-group indent unit |
| sm | 8px | `gap-2` label‚Üífield, chip internal padding |
| md | 12px | `gap-3` / `p-3` row internal padding + input padding (matches UrlTool readout rows) |
| lg | 16px | `gap-4` / `p-4` section gaps + outer tool padding (the standard tool frame) |
| xl | 24px | reserved for major vertical section breaks if needed |

Exceptions:
- **12px (`p-3`, `gap-3`) is the established intra-row rhythm** in this codebase (UrlTool readout/query rows, input padding) and is retained for visual consistency rather than snapping to 16px. This is a deliberate house value, not a violation ‚Äî it is the shipped norm across the URL/Protobuf/Base64 tools.
- Capture-group breakdown (D-08): groups indented one step (`pl-4` / 16px) beneath their parent match.
- Touch/click targets: flag toggles and chips inherit `SegmentedControl`/`CopyButton` sizing (focus-visible ring, keyboard operable) ‚Äî already WCAG-AA in the shared primitives.

## Typography

Inherited from the design system (`design/DevTools Mockup.html` + `src/index.css`). UI chrome is
IBM Plex Sans; all regex content (pattern, sample text, matches, group values, replace preview)
is **JetBrains Mono** because every character position is load-bearing for a regex tool.

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Section label | 12px | 600 (semibold) | ~1.2 | sans | Uppercase tracking-wide field labels ("PATTERN", "SAMPLE TEXT", "REPLACE", "MATCHES"), mirrors UrlTool labels |
| Body / caption | 12px | 400 (regular) | 1.4 | sans | Empty-state hint, error text, flag-help caption, "N matches" summary |
| Code / value | 13px | 400 (regular) | 1.5 | mono | Pattern input, sample-text editor, highlighted backdrop, match text, group values, replace preview, "timed out" detail |

Weights used: exactly **2** ‚Äî regular (400) for content/captions, semibold (600) for labels.
(IBM Plex 500/700 are loaded in the system but this tool uses only 400/600, matching UrlTool.)

Line heights:
- **1.5 for the mono code surfaces** ‚Äî and CRITICAL for D-01/D-02: the editable `<textarea>` and the highlight backdrop `<div>` MUST share an **identical** `line-height` (and font-family, font-size, letter-spacing, padding, border-width, white-space/wrapping) or the overlay drifts. Pin `font-mono text-[13px] leading-[1.5]` on both, identically.
- 1.2‚Äì1.4 for sans labels/captions (heading-like vs body), per the shell norm.

## Color

60/30/10 inherited from the system `@theme` (`src/index.css`). The window is dark; accent is
strictly reserved (the project's standing rule: **accent = selected/active only**, never "all
interactive"). The hero Protobuf rule "`#N` numbers neutral, accent = selected only" generalizes
here to capture-group numbers.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg-app` `#0a0b0d` (radial-gradient base) + `--color-input-bg` `#0d0f13` | Window background; pattern/sample-text/preview field surfaces |
| Secondary (30%) | `--color-win` `#15171c` / `--color-pane` `#14161b` / `--color-card`/`--color-panel` `#181b21` + `--color-bd` `rgba(255,255,255,.07)` borders | Tool frame, match-list rows, group rows, card surfaces, dividers |
| Accent (10%) | `--color-accent` `#5b9bf8`, with `--color-accent-soft` (15% tint) + `--color-accent-line` (75%) | See reserved list below |
| Destructive / error | `--color-bad` `#f0876b` | Inline error text only (invalid regex; "pattern timed out") ‚Äî see note |
| Positive | `--color-ok` `#34d399` | Reserved (not required this phase; no success-state surface needed) |

**Accent (`#5b9bf8`) reserved for ‚Äî explicit list (nothing else may use accent):**
1. The **active flag toggle** segment(s) (`g i m s u`) ‚Äî `border-accent-line bg-accent-soft text-accent` via `SegmentedControl` (D-06).
2. **Match highlight** `<mark>` background ‚Äî `bg-accent-soft` with `text-accent` (the RESEARCH ¬ßoverlay example; this is the "matches glow" of D-01).
3. **Copy-confirmed** state on `CopyButton` (the momentary "Copied" ‚Üí `border-accent-line text-accent`, inherited unchanged).
4. **Focus-visible ring** on inputs/buttons/chips (`focus-visible:ring-accent`) ‚Äî the system-wide a11y affordance, inherited.

**NOT accent (must stay neutral):**
- Capture-group **numbers** (`$1`, `$2`, ‚Ä¶) and named-group keys render in `text-tx-2`/`text-tx-3` (neutral) ‚Äî mirrors the hero's "`#N` numbers neutral" rule. Accent is for *selection*, not *enumeration*.
- The library chips (Email / URL / IPv4) are **neutral** buttons (`text-tx-2` + `border-bd`, accent only on `focus-visible`) ‚Äî they are insert actions, not a selected state.

**Destructive note:** This tool has **no destructive action** (D-11 "insert overwrites the pattern, no confirm" is non-destructive ‚Äî the field is instantly re-editable; the tool is disposable). `--color-bad` is used only as **error/timeout text color** (`role="alert"`, D-14/D-15), not for a destructive button. No confirmation dialog exists in this phase.

## Copywriting Contract

All copy below is prescriptive. Empty/error/timeout discipline mirrors Base64/Protobuf/URL
(D-13/D-14/D-15). Note: this is a paste-instant tool ‚Äî there is **no "Run"/"Test" CTA button**
(matching is live on input, the project's no-decode-button wedge). The "primary action" surfaced
to the user is the library-chip insert.

| Element | Copy |
|---------|------|
| Primary CTA (insert affordance) | Library chips labeled exactly **`Email`**, **`URL`**, **`IPv4`** (D-09/D-10). Clicking inserts that pattern+flags, overwriting the field (D-11). Chip group label (`aria-label`): **"Insert a common pattern"**. |
| Field labels | `PATTERN` ¬∑ `SAMPLE TEXT` ¬∑ `REPLACE` ¬∑ `MATCHES` (uppercase semibold labels, UrlTool style). |
| Match summary | **"{n} matches"** (e.g. "3 matches"); singular **"1 match"**; the matches section heading reads `MATCHES ({n})` mirroring UrlTool's `Query ({n})`. |
| Empty state (no pattern OR no sample text) | Neutral hint, not an error (D-13): **"Enter a pattern and some sample text to see live matches."** No error chrome, no `role="alert"`. |
| Empty replace field | No preview rendered (neutral) ‚Äî the `REPLACE`/`Preview` block is simply absent until the replace field is non-empty (D-05). No copy needed. |
| No-matches state (valid pattern, valid text, zero matches) | Neutral: **"No matches."** (mirrors UrlTool "No query parameters."). Not an error. |
| Unmatched optional group | Muted em-dash **`‚Äî`** in `text-tx-3` (D-08; mirrors UrlTool `Value` empty placeholder). |
| Error state ‚Äî invalid regex (RGX-07) | Inline `role="alert"` in `text-bad`, rest of view intact, no throw (D-14). Copy = the **native `RegExp` constructor message** surfaced verbatim (error-as-value from `buildRegex`), e.g. *"Invalid regular expression: /(/: Unterminated group"*. Do not paraphrase ‚Äî the engine's own message is the most actionable. |
| Error state ‚Äî pattern timed out (RGX-06) | Inline `role="alert"` state, UI responsive (D-15). Copy: **"Pattern timed out ‚Äî it may cause catastrophic backtracking. Try simplifying it."** Heading/marker may read **"Pattern timed out"**; the actionable detail is the second clause. |
| Replace preview label | **`PREVIEW`** label + a read-only mono output with a `CopyButton` (D-05). Caption (optional, sans 12px `text-tx-3`): **"Supports $1, $<name> and $&."** |
| Flag toggles | Single-character labels exactly: **`g` `i` `m` `s` `u`** (lowercase, mono-ish, via `SegmentedControl` button labels). Each `aria-pressed`; provide an `aria-label` per toggle, e.g. `aria-label="Toggle global flag (g)"`, `"‚Ä¶case-insensitive (i)"`, `"‚Ä¶multiline (m)"`, `"‚Ä¶dotAll (s)"`, `"‚Ä¶unicode (u)"`. Group `aria-label`: **"Regex flags"**. |
| Destructive confirmation | None ‚Äî no destructive action in this phase (see Color ¬ßDestructive note). |

**Copy tone:** terse, lowercase-where-technical, developer-direct ‚Äî consistent with the shipped
tools. No marketing voice. Error messages name the problem AND the next step (the project's
error-state discipline).

## Component / Interaction Inventory (this phase)

> Not a template section, but load-bearing for the planner/executor. Each maps to a requirement.

| Component | Built from | Requirement | Notes |
|-----------|-----------|-------------|-------|
| Pattern field | `<input>`/`<textarea>` mono, UrlTool input styling (`border-bd bg-input-bg focus-visible:ring-accent`) | RGX-01/03 | `aria-invalid` on error |
| Flag toggles `g i m s u` | `SegmentedControl`-style row of 5 independent `aria-pressed` buttons (D-06) | RGX-03 | `g` toggles replace all-vs-first ONLY; enumeration is always g-forced internally (D-07) |
| Library chips | Row of 3 neutral `<button>` chips above pattern (D-10) | RGX-05 | Insert overwrites, no confirm (D-11); patterns are simple/linear (D-12) |

| Match list + capture-group breakdown | Per-match rows; numbered + named groups indented beneath, each with `CopyButton` (D-08) | RGX-01/02 | Group numbers/keys NEUTRAL (not accent); unmatched optional group `‚Äî` |
| Replace field + preview | `REPLACE` input + read-only `PREVIEW` mono output + `CopyButton` (D-05) | RGX-04 | Hidden when replace field empty; native `replace` ($1/$<name>/$&) |
| Error / timeout banner | Inline `role="alert"` `text-bad`, rest of view intact | RGX-06/07 | error-as-value; never a throw |

**Layout (D-04):** ONE combined scrolling view, NO mode switch ‚Äî chips ‚Üí pattern ‚Üí flags ‚Üí
sample-text editor ‚Üí matches+groups ‚Üí replace+preview, all live at once. Layout-agnostic
(responsive Tailwind, `min-w-0`, no fixed widths; layout chrome lives in the shell).
StatusBar/byteCount: **omit** (regex is not a byte transform; Phase 8 opt-in precedent).

## Accessibility Notes (WCAG-AA, phase-boundary gate)

- Phase boundary requires a passing `gsd-ui-review` WCAG-AA audit + `tauri build` human walkthrough.
- All accent-on-tint text combinations are pre-cleared: `--color-accent #5b9bf8` was deliberately brightened to clear AA on `accent-soft` (`src/index.css` comment, 03-UI-REVIEW #2). The match `<mark>` (`text-accent` on `bg-accent-soft`) inherits this cleared pairing.
- Flag toggles + chips are keyboard-operable with `focus-visible:ring-accent` (inherited from `SegmentedControl`/`CopyButton`).
- Copy is never hover-gated (project rule) ‚Äî every `CopyButton` is a visible, focusable button.
- Error/timeout states use `role="alert"` so they are announced.
- **D-02 alignment is also an a11y/focus hazard** ‚Äî the overlay must keep the real `<textarea>` focusable and the caret visible; verify caret + scroll alignment on the **real WKWebView**, not just Chromium.
