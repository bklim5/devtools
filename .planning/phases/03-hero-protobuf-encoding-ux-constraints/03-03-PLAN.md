<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pasted text/base64/hex → bytes | All three input fields are untrusted user input |
| tool → OS clipboard | Copy writes cross into the OS via the platform seam |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering/Injection | invalid base64/hex input | mitigate | All conversion routes through `bytes.ts`, which validates (hex length/charset throw; base64 via native/atob). Invalid input becomes an explicit per-field error string, never a silent corruption (D-13). |
| T-03-07 | Injection | decoded text rendered in textareas + copied | mitigate | Values render as React-controlled `<textarea value>` (text node, escaped); copy writes the plain string via `platform.clipboard.writeText`. No HTML injection surface; no `dangerouslySetInnerHTML`. |
| T-03-08 | Elevation of Privilege | clipboard capability | mitigate | Copy uses `platform.clipboard` only; no `@tauri-apps/*` import in the tool (grep-verified). |
| T-03-09 | DoS | very large paste | accept | Offline single-user tool; bytes.ts conversions are linear; no nested recursion. A multi-MB paste is the user's own input on their own machine. |
</threat_model>
