<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argv → driver | Maintainer-supplied tokens (only `--dry-run` accepted); rejected loudly otherwise |
| build output (`.sig` glob) → latest.json | The single most security-critical link: the signature that every install trusts is derived from a globbed file — wrong/stale = unverifiable update |
| process env → presence checks | Signing key + Apple creds; their PRESENCE is read, their VALUE must never be returned/logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | `assertSingleSig` | mitigate | Throw on 0 (signing-env hint) AND >1 (ambiguous) so a stale/wrong `.sig` can never silently flow into `latest.json`; unit-tested both branches |
| T-11-02 | Tampering | `parseLipoArchs` | mitigate | Require BOTH `x86_64` and `arm64`; arm64-only/arm64e returns false — proves the universal artifact is genuinely fat, not a single-arch build mislabeled |
| T-11-03 | Tampering / DoS | `buildAssetUrl` | mitigate | Hard-code `bklim5/devtools-releases`; unit test asserts the private `bklim5/devtools` slug never appears — the updater URL can't point at the wrong (private/unreachable) repo |
| T-11-04 | Information disclosure | `hasSigningEnv` / `hasAppleEnv` | mitigate | Return BOOLEANS only; unit test asserts the return value is `true`/`false`, never the secret string — no path for a value to leak through the pure core |
| T-11-05 | Tampering | `assertVersionMatches` | mitigate | Pure equality check feeding the REL-12 post-publish guard; mismatch throws printing both versions — a wrong-version `latest.json` is caught |
</threat_model>
