# Phase 9: Pure release core + housekeeping - Context

**Gathered:** 2026-06-02
**Status:** Ready for planning

<domain>
## Phase Boundary

The only logic that can silently corrupt a release — version-bump math, per-manifest version edits, and `latest.json` assembly — exists as **pure, deterministic functions** in a new `src/lib/release/` (`version.ts` + `manifest.ts`), fully unit-tested by the **existing** `vitest`/`tsc`/`eslint` gate (no new wiring). Plus two one-time housekeeping fixes: reconcile the drifted `Cargo.toml` `[package].version` (`0.1.0` → current `0.2.1`) and confirm the stale `latest.json` is untracked + gitignored.

**Explicitly NOT in this phase:** no I/O, no scripts, no git/network actions, no app UI. Those land in Phase 10 (`bump-and-tag` driver, wires `version.ts` to real I/O) and Phase 11 (`build-and-publish` driver, wires `manifest.ts`).

</domain>

<decisions>
## Implementation Decisions

### Manifest editing (`version.ts`)
- **D-01:** Version edits are **surgical content/string replacements** for all three manifests — NOT JSON.parse→stringify round-trips. Each `setXVersion(content, newVersion): string` is a pure string→string transform that rewrites only the version line, preserving all surrounding formatting, key order, and indentation. This keeps the eventual bump commit diff to single lines and is the only safe way to leave Cargo dependency `version = "…"` pins untouched.
- **D-01a (robustness):** The match must anchor precisely: the **first top-level `"version"`** key in `package.json` / `tauri.conf.json`, and the `version` line under the **`[package]`** section in `Cargo.toml` (never dependency-section versions). The transform must **fail loudly on 0 or >1 matches** so a malformed/unexpected file can't silently no-op.

### `bumpSemver` (`version.ts`)
- **D-02:** Minimal, hand-rolled `MAJOR.MINOR.PATCH` only (no `semver` dependency — holds the zero-new-deps line). Standard rollover: patch `0.2.1→0.2.2`; minor `→0.3.0` (patch reset); major `→1.0.0` (minor+patch reset).
- **D-02a:** **Throw a clear error** on malformed input or unknown bump level. No prerelease / build-metadata support — the app has never used them (YAGNI).
- Single computed source of truth for the next version (success criterion #2): bump once, reuse everywhere.

### `manifest.ts` — `buildLatestJson` + `platformKey`
- **D-03:** `manifest.ts` is **PURE** — no `Date.now`, no filesystem. This is deliberate and load-bearing: it makes the corruption-prone manifest assembly unit-assertable by the existing vitest gate with zero mocks (success criterion #2). All time/I/O-dependent inputs are **injected by the caller** (Phase 11's `build-and-publish.mjs` globs the fresh `.sig`, stamps `pub_date = now`, computes the URL, then calls this function and writes the result).
- **D-04:** Signature is a **single options object**: `buildLatestJson({ version, pubDate, url, signature, notes? })` → returns the full `latest.json` object. Named args avoid positional mistakes (esp. the two adjacent `url`/`signature` strings) and are trivially extensible.
- **D-04a:** `notes` is **optional, defaults to `""`**. The updater UI tolerates empty notes; Phase 11 can pass real notes later without the pure core forcing it.
- **D-05:** Emits **both** `darwin-aarch64` and `darwin-x86_64` platform keys (via the `platformKey` helper), each with the **same URL and same signature** (one universal artifact serves both arches). **No invented `darwin-universal` key.** Output object shape matches the existing on-disk `latest.json` (`version`, `notes`, `pub_date`, `platforms`).

### Housekeeping
- **D-06:** **`latest.json` untrack is idempotent / verify-only.** Repo state already has it untracked (`git ls-files latest.json` is empty) and `/latest.json` is already gitignored. Run `git rm --cached latest.json` **only if** it shows as tracked (currently a no-op), and assert the gitignore entry exists. Verify against actual repo state at execution — don't assume tracked. **Leave the leftover on-disk 0.2.1 `latest.json`** in the working tree (gitignored, harmless, regenerated by Phase 11).
- **D-07:** **Cargo reconcile dogfoods `setCargoVersion`.** Perform the one-time `0.1.0 → 0.2.1` reconcile by running the new `setCargoVersion` function on the real `src-tauri/Cargo.toml` (not a blind hand-edit), proving the `[package]`-scoped edit works on the actual file and leaves dependency pins untouched.

### Conventions / layout
- Follow the established `src/lib/format/` pattern: co-located source + test (`version.ts` + `version.test.ts`, `manifest.ts` + `manifest.test.ts`) under new `src/lib/release/`.
- Zero new runtime **and** dev dependencies (Node builtins suffice; the converged research finding is that even devDeps are unnecessary here).

### Claude's Discretion
- Exact function names beyond those named above, internal helper decomposition, and test-case naming.
- Whether the `setCargoVersion` dogfood reconcile (D-07) happens via a one-off script invocation, a `tsx` snippet, or simply asserting the post-edit content in a test fixture that also writes the file — any approach that proves the function ran against the real file is acceptable.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- **`src/lib/format/` layout** — co-located `name.ts` + `name.test.ts`, pure transforms, no I/O. Directly mirror this for `src/lib/release/`.
- **Existing vitest/tsc/eslint gate** — covers `src/lib/**` automatically; new release files are picked up with zero new config (this is why the phase is "auto-covered").

### Established Patterns
- **Zero-dependency pure-logic ethos** — every tool so far (decoder, formatters, bytes, time) is hand-rolled over native APIs. `version.ts`/`manifest.ts` continue this: hand-rolled semver + plain object assembly, no `semver`/`toml`/`json5` libs.
- **Pure logic in `src/lib/`, I/O in thin callers** — formatters keep transforms pure and route side effects through the platform seam; release core keeps transforms pure and routes I/O to the Phase 10/11 `.mjs` drivers.

### Integration Points
- **Phase 10** imports `version.ts` (`bumpSemver` + the three `setXVersion` editors) to do the lockstep 3-file write.
- **Phase 11** imports `manifest.ts` (`buildLatestJson`/`platformKey`) to generate `latest.json` from the fresh `.sig`.
- Reconcile target: `src-tauri/Cargo.toml` `[package].version` (`0.1.0` → `0.2.1`).

### Verified repo state (2026-06-02, confirm at execution)
- `package.json` = `0.2.1`, `src-tauri/tauri.conf.json` = `0.2.1`, `src-tauri/Cargo.toml` `[package].version` = `0.1.0` (the drift).
- `git ls-files latest.json` → **empty** (already untracked). `.gitignore` line 11 = `/latest.json`. On-disk `latest.json` (694 B, 0.2.1) present but gitignored.
