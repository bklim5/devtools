/** D-10/D-11 render-overlay reconciliation. registryIds is the canonical
 *  ENABLED_TOOLS order (single control plane). savedOrder is UNTRUSTED (hand-edited
 *  prefs.json) — drop non-strings, de-dupe, drop ids not in the registry, then append

Implementation notes:
- reconcileToolOrder: build a `Set` of registryIds for O(1) membership; iterate savedOrder

  their original order.
- moveToolInOrder: find the current index; if not found return `order` (or a shallow copy)
  unchanged; else splice it out, clamp `toIndex` to `[0, arr.length-1]` of the post-removal
  array, splice it back in. Always return a fresh array.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| prefs.json (on disk) → app load | `toolOrder` is read from a user-writable/hand-editable preferences blob — UNTRUSTED input crosses here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-01 | Tampering | `mergePreferences` reading `toolOrder` from prefs.json | mitigate | `coerceToolOrder` accepts only string members and de-dupes; non-strings/objects/numbers dropped. A non-array `toolOrder` yields `[]`. No throw → no blank app. |
| T-16-02 | Denial of Service | Oversized/duplicate-stuffed `toolOrder` (e.g. 10k entries) | mitigate | `reconcileToolOrder` gates membership against the registry `Set`, so output length is bounded by the registry (11 tools) regardless of saved-blob size; duplicates collapse. Oversized junk cannot bloat the rendered list. |
| T-16-03 | Tampering | `toolOrder` referencing unknown/removed IDs | mitigate | `reconcileToolOrder` drops ids not in `registryIds` (D-11) — never renders or routes to a phantom tool; registry stays the single control plane. |
| T-16-04 | Injection | tool IDs flowing toward DOM/aria-live text (Plan 02) | accept (here) / mitigate (Plan 02) | IDs are registry-controlled, not user-supplied free text; coercion keeps them as opaque strings. The aria-live message in Plan 02 uses the registry `name`, not raw stored input. No injection surface in this plan (no DOM). |
</threat_model>
