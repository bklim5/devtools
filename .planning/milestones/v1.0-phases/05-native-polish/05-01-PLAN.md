<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| second process ŌåÆ running app | The single-instance callback receives argv/cwd from an arbitrary second launch ŌĆö untrusted input. |
| Tauri capability allowlist ŌåÆ plugin commands | Every plugin command is gated by `capabilities/default.json`; over-granting is privilege escalation. |
| app ŌåÆ network | Offline-by-design; any new plugin opening a connection breaks the core constraint. |
| build profile ŌåÆ release artifact | The debug-only webdriver server must never compile into a release bundle. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | capabilities/default.json grant scope | mitigate | Grant ONLY `global-shortcut:allow-register/unregister/is-registered` + `window-state:default`; reuse existing `core:default` for window control (A2); no wildcard `core:window` or `global-shortcut:default`. |
| T-05-02 | Tampering / Spoofing | single-instance callback argv/cwd | mitigate | Callback ignores `_argv`/`_cwd` entirely (underscore-prefixed); it only shows/focuses the existing window. Any future deep-link MUST route through the existing `getToolById`/`resolveStartupTool` validation (ENABLED_TOOLS only) ŌĆö not raw argv. |
| T-05-03 | Information Disclosure | new plugins opening network connections | mitigate | single-instance / global-shortcut / window-state make no network calls; `security.csp` (`connect-src 'self' ipc: http://ipc.localhost`) stays unchanged; offline-by-design holds. |
| T-01-10 / T-01-11 | Elevation of Privilege | webdriver server leaking into release | mitigate | The `#[cfg(all(debug_assertions, feature = "webdriver"))]` double-gate stays verbatim; re-verify `cargo tree --release \| grep -c webdriver` = 0 after the lib.rs edit (Pitfall 7). |
</threat_model>
