| WebDriver server (:4445) ↔ network | The embedded W3C server is a remote-control surface; must exist ONLY in debug builds bound to localhost |
| automation harness ↔ app | The UI-gate driver becomes the trusted verifier for all later phases — its correctness gates everything downstream |
| vite-preview fallback ↔ browser | The fallback serves the static bundle outside Tauri; the platform seam's browser impl (navigator.clipboard) is the surface exercised there |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-01-11 | Spoofing | WebDriver server bound to a routable interface lets a remote actor drive the app | mitigate | Bound to `127.0.0.1:4445` (localhost only); never `0.0.0.0`; wdio.conf targets 127.0.0.1 and the bind is verified/forced; debug-only |
| T-01-12 | Repudiation | Automation path decision undocumented → later phases can't reproduce the UI gate | mitigate | Chosen path + rationale + the exact reproducible command (`bash scripts/e2e-spike.sh`) recorded in docs/phase-0-notes.md (HRN-02 deliverable, D-03) |
| T-01-16 | Information disclosure | The vite-preview fallback runs the app outside Tauri; navigator.clipboard in a non-secure context could silently fail or leak | accept | Fallback is a dev-only verification surface (localhost vite preview), not shipped; the browser clipboard impl guards availability; no untrusted data |
