    decodeJwt.test.ts:
    - happy path: a valid 3-segment token decodes to { kind:"ok", header: <object>, payload: <object>, signature: <raw string>, alg: <header.alg> } with header+payload as parsed JSON
      (use a known token e.g. header {"alg":"HS256","typ":"JWT"} payload {"sub":"1234567890","name":"John Doe","iat":1516239022})
    - empty input → { kind:"empty" }, NOT an error
    - "a.b" (2 segments) → { kind:"error", scope:"token" } message mentions segment count / "3"
    - a token whose header segment is non-base64url (e.g. contains "!!!") → { kind:"error", scope:"header" }
    - a token whose payload decodes to non-JSON (base64url of "not json") → { kind:"error", scope:"payload" }
    - decode NEVER throws past the boundary (returns a result/error object), no crash on adversarial input
  </behavior>
  <action>
    Create `src/tools/jwt/decodeJwt.ts` (pure, imports only `@/lib/bytes`):
    - `decodeJwt(token: string): DecodedJwt`, a discriminated union:
      `{ kind: "empty" }` | `{ kind: "error"; scope: "token"|"header"|"payload"; message: string }` |
      `{ kind: "ok"; header: unknown; payload: unknown; signature: string; alg?: string }`.
    - Algorithm: trim; empty → `{kind:"empty"}`. `const parts = token.split(".")`. If `parts.length !== 3` → `{kind:"error", scope:"token", message:`Expected 3 dot-separated segments, got ${parts.length}`}`.
    - For header (parts[0]) and payload (parts[1]): `bytesToUtf8(base64ToBytes(part, "base64url"))` inside try/catch → on throw return `{kind:"error", scope, message:"Segment is not valid base64url"}`; then `JSON.parse` inside try/catch → on throw return `{kind:"error", scope, message:"Segment is not valid JSON"}`.
    - signature = parts[2] shown RAW (no decode). alg = `(header as Record<string,unknown>)?.alg` if a string.
    - Wrap so it NEVER throws past the boundary (T-04-07). Do NOT verify the signature, do NOT accept a key (D-09).
    Create `src/tools/jwt/decodeJwt.test.ts` covering <behavior>. Build fixtures by base64url-encoding known JSON (use `@/lib/bytes` bytesToBase64 with "base64url", or inline literal segments).
  </action>
