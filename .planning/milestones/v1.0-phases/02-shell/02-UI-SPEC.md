# Phase 2 — UI Design Contract

> Visual and interaction contract for the application shell (compact sidebar + ⌘K command palette + theme/accent). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Authoritative visual source:** `design/DevTools Mockup.html` (CSS tokens + `.sidebar`/`.navitem`/`.palette` rules). All values below are derived from it, not invented. Tokens are already vendored in `src/index.css` (`@theme` block) — this phase consumes them, it does not re-define the palette.
>
> **Scope:** shell chrome only — titlebar, compact sidebar, ⌘K palette, status of the active tool surface. Tool-content UIs (Protobuf tree, encoding panes, etc.) are Phase 3/4 and are out of scope; in Phase 2 each enabled tool renders a shared "coming in Phase 3" placeholder (D-01).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (bespoke CSS-variable system, vendored from the canonical mockup) |
| Preset | not applicable |
| Component library | none — hand-built React components against the mockup. **shadcn/cmdk/fuse.js deliberately rejected** (D-06 + zero-runtime-dependency + offline ethos; cmdk's DOM/focus model fights the bespoke palette markup) |

## Spacing Scale

The mockup uses a near-4-point rhythm with a few mockup-specific sidebar/palette values. Declared scale (multiples of 4 — the planner/executor default to these; the mockup-specific exceptions below are the only permitted off-grid values, all traced to `design/DevTools Mockup.html`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, accent-bar width band |
| sm | 8px | Light-light element spacing, lights gap, palette-list padding |
| md | 12px | Sidebar inner gap, navitem icon↔name gap, palette item gap |
| lg | 16px | Sidebar padding, palette-search padding |
| xl | 24px | Reserved (tool-content padding lands Phase 3) |
| 2xl | 32px | Layout-level gaps |
| 3xl | 48px / 64px | Major breaks (not used by shell chrome) |

**Exceptions (mockup-specific, off the 4-grid — permitted, sourced from the mockup, do not "round"):**
- Titlebar height `44px`; status bar height `38px`; sidebar search box height `38px` (`.titlebar`, `.statusbar`, `.search`).
- Sidebar width `268px` (compact = icon + name; D-02). No icons-only/full modes in v1 (deferred).
- `navitem` padding `8px 11px` (compact), `navbar-accent` width `3px`, height `56%`.
- Palette panel `min(560px, 92vw)`, scrim padding-top `14vh`, list `max-height: 340px`, palette-item padding `9px 10px`.
- Border radii: window `13px`, navitem `9px`, palette `14px`, palette item `8px`, search box `9px`, kbd `5px`.

Touch targets are not a concern (macOS desktop, pointer + keyboard). Keyboard focus targets must remain visibly reachable (UX-04).

## Typography

Three UI sizes + one mono size for chrome (the mockup uses more sizes across tool content, but shell chrome resolves to these). Two weights only: 500 (medium) and 600 (semibold); 400 is the document body default carried from `index.css`. Sourced from `design/DevTools Mockup.html`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Heading / title | 19px | 600 | 1.2 | Tool header `h1` (`.th-left h1`). Titlebar app title is 13px/600 (a label, see below) |
| Body | 14px | 500 | 1.5 | Palette item name (`.pname` 14px/500), palette input 16px/regular |
| Label | 13px–13.5px | 600 | 1.3 | Sidebar nav name (`.navname` 13.5px/600), titlebar title (13px/600), search input 13px |
| Caption / uppercase | 11px | 600 | 1.35 | Section labels (`.pglabel`, `.pane-label`): uppercase, letter-spacing ~0.09em, color `--tx-3` |
| Mono (chrome) | 11.5px | 400/500 | 1.35 | `kbd` shortcut hints, status-bar text (`--font-mono`) |

**Rules:**
- UI text is IBM Plex Sans; only `kbd`, byte/code, and status-bar metrics use JetBrains Mono (`--font-mono`).
- Exactly two declared chrome weights (500, 600); document default 400 is the unstyled baseline. Do not introduce 700 in shell chrome (the mockup loads 700 but reserves it for tool content).
- Palette search input renders at 16px (prevents zoom/feels native) — this is the one body-size exception and is intentional (`.palette-search input`).

## Color

Dark-only for v1 (D-10), but the theme is stored as a **named value** (`"dark"`, not a boolean) and all color flows through CSS variables so a light theme can be added later with no component rework. The 60/30/10 split maps to the mockup's surface hierarchy.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg-app` `#0a0b0d` (radial-gradient to `#14161c`) + `--pane` `#14161b` + `--win` `#15171c` | App background, window body, main tool pane |
| Secondary (30%) | `--sidebar` `#101216`, `--titlebar`/`--statusbar` `#1b1e24`, `--panel` `#181b21`, palette panel `#1a1d23` | Sidebar, titlebar, status bar, palette surface, cards |
| Accent (10%) | `--accent` `#3b82f6` (+ `--accent-soft` = accent@15%, `--accent-line` = accent@75%) | **Selected/active state only** — see reserved list |
| Destructive | not used in Phase 2 | No destructive actions in the shell (no delete/reset-with-data-loss surface). `--bad` `#f0876b` exists for tool-content error text (Phase 3+), not shell chrome |

**Accent reserved for (explicit — never "all interactive elements"):**
- The **active sidebar item**: left `--accent` bar (`.navbar-accent` scaleY 0→1), `--accent-soft` background tint, and active icon recolored to `--accent` (`.navitem.on`). Project-wide rule: **accent = selected/active only**.
- The **highlighted palette row** (keyboard cursor / hover): `--accent-soft` background, icon in `--accent` (`.pitem.on` / `.pitem:hover`).
- The persisted **accent-color preference** itself (D-10): the mockup exposes a small swatch set — `#3b82f6` (default), `#64748b`, `#10b981`, `#8b5cf6`, `#e0a42b`. Changing it re-drives `--accent` via the CSS-variable system; `--accent-soft`/`--accent-line` derive automatically through `color-mix`.

**Text tiers (all on dark):** `--tx` `#e7e9ee` (primary), `--tx-2` `#989da7` (secondary), `--tx-3` `#686d77` (tertiary/placeholder/captions). Borders: `--bd` `rgba(255,255,255,0.07)`, `--bd-2` `rgba(255,255,255,0.12)`.

**WCAG-AA note (UX-04, phase-gate audit):** Phase 1's audit already bumped muted text from white/40 → white/60. Hold the line: body/label text must clear AA contrast on its surface; disabled/inactive state must **not** be signalled by opacity alone (the mockup's `.navitem.pro { opacity:.62 }` premium-dim pattern is **out of scope** for Phase 2 — no premium tooling here, so it does not apply). Focus indicators must be visible for sidebar `NavLink`s and the palette input.

## Copywriting Contract

The shell has no data-entry form, so the classic CTA/empty/error matrix maps to shell-specific surfaces. All copy below is the locked contract.

| Element | Copy |
|---------|------|
| Primary CTA (palette open affordance) | `⌘K` hint pill in the tool header (the palette is the no-mouse switch path, D-03/D-07). Label/affordance reads as a keyboard hint, not a button verb — surface the `⌘K` kbd, optionally with "Search tools". |
| Palette input placeholder | `Search tools…` (palette searches tools only in v1 — D-04) |
| Palette section label — recents | `RECENT` (uppercase `.pglabel`; shown first when query is empty, ~5 items most-recent-first — D-05) |
| Palette section label — all tools | `ALL TOOLS` (uppercase `.pglabel`; remaining tools in registry order after recents — D-05) |
| Empty state (palette no-match) | Quiet single row: `No tools match`. **Never an error** — neutral `--tx-3`, no error color, no icon-of-doom (D-07). |
| Empty state (first run / no recents) | Not surfaced — the app opens **directly into a tool** (Protobuf hero on first run, D-12), there is no "pick a tool" screen (SHL-06). Palette empty-query simply shows `ALL TOOLS` with no `RECENT` group until tools have been used. |
| Tool placeholder (D-01) | Shared placeholder component for enabled-but-unbuilt tools. Heading = the tool's `name` from the registry; body = a single quiet line, e.g. `Coming in Phase 3` (Protobuf/Base64) or the appropriate phase. Tone: calm, informational, `--tx-3`; no spinner, no "under construction" cliché. |
| Error state | None in the shell. A tampered/removed `lastUsedId` from the store silently falls back to the hero (D-13) — no error surfaced to the user. A bad `#/tools/<id>` deep-link validates via `getToolById` and falls back (D-14) — no 404, no error toast. |
| Destructive confirmation | None — Phase 2 has no destructive action. (Reset-to-defaults, if built under Claude's discretion, writes default values; it is not a data-loss action and needs no confirmation dialog.) |

**Footer shortcut hints (palette `.palette-foot`):** show keyboard affordances using `kbd` — e.g. `↑↓ navigate`, `↵ open`, `esc close`. Mono font, `--tx-3`, 11px.

## Interaction Contract (shell-specific)

Not in the base template but load-bearing for this phase — the checker/executor need these.

| Surface | Interaction | Contract |
|---------|-------------|----------|
| Sidebar | Pointer + standard Tab focus only | No arrow/j-k nav, no ⌘1..⌘6 (D-03). Each item is a `NavLink`; active state from `isActive`. The palette is the sole keyboard switch path. |
| ⌘K palette — open | `(metaKey OR ctrlKey) && key==='k'` toggles; also a click affordance in the header | Never auto-opens on launch (D-07). `preventDefault` on the combo. |
| ⌘K palette — navigate | `↑`/`↓` move a highlighted index (the `.pitem.on` accent row); `Enter` navigates to the highlighted tool and closes; `Esc` closes | Wraps within the visible (recents + all) list. Highlight is keyboard-driven; hover mirrors the same `--accent-soft` styling. |
| ⌘K palette — filter | Typing fuzzy-ranks across name + keywords + description (in-house ranker, D-06) | Empty query → `RECENT` (≤5) then `ALL TOOLS`; non-empty → ranked flat list; no match → quiet `No tools match` row. |
| Active tool surface | Renders the routed `<Outlet/>` (placeholder in Phase 2) | Tool components stay layout-agnostic (no fixed widths; responsive) — layout chrome lives only in the shell (UX-05). |
| Startup | Index route resolves via single `resolveStartupTool()` | Precedence: explicit `#/tools/<id>` (valid+enabled) > last-used (valid, else hero) > hero `protobuf-decoder` (D-12/13/14). No "pick a tool" step. |
| Theme/accent | Persisted via the `Store` seam; applied through CSS variables | Dark-only `"dark"` named value + accent swatch (D-10). Re-driving `--accent` cascades to `--accent-soft`/`--accent-line`. |
