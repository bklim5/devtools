Output: `usePreferences`, `useRecentTools`, `resolveStartupTool` + `HERO_TOOL_ID`, router index-route wiring, prefs preloaded in main.tsx (Pitfall 3).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted prefs ↔ hooks | `lastUsedId`/`recentToolIds`/`theme`/`accent` read from the store are untrusted input |
| URL hash (deep-link) ↔ router | `#/tools/<id>` is attacker/user-controllable and crosses into navigation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | resolveStartupTool / router index | mitigate | Validate the `#/tools/<id>` deep-link target via `getToolById` before navigating (V5); unknown ids fall back to hero — never renders an arbitrary route (Task 2 + Task 3) |
| T-02-08 | Tampering | persisted prefs (lastUsedId, recents, theme, accent) | mitigate | Treat store values as untrusted: validate `lastUsedId` against the registry (D-13 fallback), merge prefs over DEFAULT_PREFERENCES accepting only known fields/types, ignore non-string recent ids (Task 1 + Task 2) |
| T-02-09 | Tampering | accent color value from store | accept | Accent is applied as a CSS variable value only (no eval/injection sink); a malformed value simply fails to render a color — low risk, single-user local app |
</threat_model>
