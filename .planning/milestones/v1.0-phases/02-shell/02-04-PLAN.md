<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user keyboard/pointer ↔ shell | ⌘K, typing, ↑/↓/Enter/Esc and clicks are user input into the palette/sidebar |
| palette query ↔ rankTools | Free-text filtered against the registry; pure string processing |
| recents/last-used (store) ↔ palette/sidebar | Persisted ids read back are untrusted (validated via getToolById before use) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | CommandPalette recents rendering | mitigate | Recent ids from the store are mapped through `getToolById` and unknown ids are skipped — a tampered recents list can never render or navigate to an arbitrary route (Task 2) |
| T-02-11 | Information Disclosure | shell chrome | accept | No secrets/PII in Phase 2; the shell renders only tool names/placeholders — nothing sensitive to disclose |
| T-02-12 | Tampering | palette query → navigation | mitigate | Enter navigates only to ids drawn from ENABLED_TOOLS (rankTools operates over the registry, not arbitrary input) — the query cannot synthesize a route id (Task 2) |
</threat_model>
