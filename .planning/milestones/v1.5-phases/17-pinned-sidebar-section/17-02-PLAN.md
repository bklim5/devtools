<objective>
Extend the v1.4 reorderable `Sidebar.tsx` into a two-group pinned/unpinned sidebar, consuming
the Plan 17-01 backbone (`partitionTools`, `togglePinned`, `setPinnedToolIds`). No new files
except the e2e extension; no new deps. The registry array, ⌘K palette, and router are NEVER mutated.

Six wired behaviors:
1. **Two-group render** — replace the single `orderedIds` line with
   `const { pinned, unpinned } = partitionTools(preferences.pinnedToolIds, preferences.toolOrder, registryIds)`;
   render the pinned group (SR-named "Pinned tools") above the unpinned list with a bare neutral
   divider between them, shown ONLY when `pinned.length > 0` (D-15, PIN-03).
2. **Pin icon** LEFT of the grip handle (grip stays outermost) — pinned rows show a persistent
   FILLED pin (the unpin click target); unpinned rows show an OUTLINE pin on hover + focus-visible
   only, mirroring the grip's reveal; neutral tokens; click toggles, does NOT navigate (D-14, PIN-01/02/04).
3. **Alt+P** on the focused handle toggles the row's tool, announced "Pinned {name}" / "Unpinned {name}"
   via the existing aria-live region using the registry `tool.name` (D-13, PIN-05).
4. **Per-group reorder** — drag + Alt+↑/↓ run independently within each group; pinned reorders mutate
   `pinnedToolIds` (via `setPinnedToolIds(moveToolInOrder(pinned, …))`), unpinned reorders mutate
   `toolOrder` (as today); no cross-boundary drag (D, PIN-06).
5. **"Unpin all"** as a second item in the existing right-click "Reset order" menu (reusing the
   Shift+F10/ContextMenu entry), shown only when `pinned.length > 0`, calling `setPinnedToolIds([])` (D-16, PIN-09).
6. **Focus management** — keep focus on the toggled/moved tool's handle across the re-render by reusing
   the shared `focusAfterMoveRef` + `handleRefs` (one map across BOTH groups).

Plus: EXTEND `test/e2e/sidebar.e2e.ts` with the keyboard path (Alt+P, Alt+↑/↓ within a group, "Unpin
all", persistence) on the real WKWebView. Native pointer drag-within-group is manual-walkthrough only.
