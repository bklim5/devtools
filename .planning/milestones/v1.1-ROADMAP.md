# Milestone v1.1: Formatters

**Status:** ✅ SHIPPED 2026-06-02
**Phases:** 7–8
**Total Plans:** 4

## Overview

v1.0 shipped the six-tool, distributable, self-updating macOS app (Phases 1–6, signed off 2026-06-01). **v1.1 "Formatters"** adds the first two post-v1.0 tools — a **JSON formatter** and an **XML formatter** — both implemented **zero-runtime-dependency** over native browser APIs (`JSON`, `DOMParser`/`XMLSerializer`), in a shared two-pane paste-instant `FormatterView` that follows the existing tool patterns exactly. A small sibling cleanup makes the shared `StatusBar` byte-count readout **opt-in** so it only appears where input/output size is meaningful.

The work split along the boundary the approved design spec prescribed (`docs/superpowers/specs/2026-06-02-json-xml-formatters-design.md`): **Phase 7** delivered the `FormatterView` + both formatter tools (FMT-01..08); **Phase 8** shipped the opt-in `StatusBar` cleanup (UIX-01) once the Formatters were consuming `StatusBar`, so the keep/drop decision was made against a complete, real set of callers. Every phase was gated by the standing binding harness — per-task DoD (`/simplify` → `/codex:review` → `vitest`+`tsc`+`eslint` green → real-WKWebView UI verification) and a phase-boundary human sign-off on a fresh `tauri build` + a passing `gsd-ui-review` WCAG-AA audit. The hero decoder (`src/lib/protobuf/decoder.ts`) and its 19 tests stayed byte-for-byte untouched; no new runtime dependencies; HashRouter only; offline-by-design held.

## Phases

### Phase 7: Formatters

**Goal**: Two new tools — `json-formatter` and `xml-formatter` under `category: "formatting"` — ship into the registry-driven shell behind a shared two-pane paste-instant `FormatterView`, giving the user a jsonlint-style validate/prettify/minify (+ JSON sort-keys) experience entirely offline with zero new runtime dependencies.
**Depends on**: Nothing within v1.1 (builds on the shipped v1.0 shell, registry, `StatusBar`, `CopyButton`/`useCopyFeedback`, and platform clipboard seam)
**Requirements**: FMT-01, FMT-02, FMT-03, FMT-04, FMT-05, FMT-06, FMT-07, FMT-08
**Plans**: 3 plans

Plans:

- [x] 07-01: shared-foundation — promote ResizableSplit to shared, additive StatusBar byte-delta, shared FormatResult type
- [x] 07-02: json-formatter — pure formatJson + shared FormatterView + json-formatter tool + registry + e2e
- [x] 07-03: xml-formatter — pure formatXml (DOMParser) + xml-formatter tool + registry + e2e

**Details:**

**Success Criteria** (all TRUE at sign-off):
1. Pasting JSON validates instantly (no format button) and renders the prettified result; invalid JSON clears the output pane and surfaces the error as **line:col + message** in the status bar, and empty input shows an "empty" status (not an error) — FMT-01. ✓
2. The JSON tool's toolbar drives the output live: selectable indent **2 / 4 / tab** (FMT-02), a **minify** toggle that compresses to a single line (FMT-03, minify wins over prettify when on), and a **sort-keys** toggle that recursively sorts object keys while preserving array order (FMT-04). ✓
3. Pasting XML validates well-formedness instantly via `DOMParser`; a `<parsererror>` surfaces its message (with line where the engine provides one) and clears the output (FMT-05); prettify re-emits with the selected indent **preserving comments, CDATA, attributes, and processing instructions** + the `<?xml?>` declaration (FMT-06); and minify strips insignificant inter-element whitespace (FMT-07). ✓
4. Each tool's output pane exposes a **visible, keyboard-focusable copy control** reachable in ≤1 keystroke (no hover-only), copying through the platform clipboard seam (FMT-08). ✓
5. Both tools appear in the sidebar, ⌘K palette, and router solely by appending to the `TOOLS` array (single control plane, no other wiring); they are layout-agnostic; the pure formatters live in `src/lib/format/` and are independently unit-tested (TDD); and the phase ended with the decoder's 19 tests still green, a passing `gsd-ui-review` WCAG-AA audit, and human sign-off on a fresh `tauri build`. ✓

**Outcome:** Executed across 3 sequential waves (07-01 → 07-02 → 07-03). Phase verification 5/5 must-haves. Code review surfaced 3 warnings — all fixed TDD (WR-01 XML lost the `<?xml?>` declaration + doc-level comments/PIs; WR-02 timing chip measured a state setter not the format pass; WR-03 first-match `indexOf` mislocated the V8 error offset). UI review PASSED WCAG-AA (18/24, no blockers); its 3 advisory findings fixed too. Real-WKWebView e2e: **10/10 specs green** on WebKit 605.1.15. The live gate caught a real regression unit tests missed — real WebKit concatenates `<parsererror>` text with no newlines, so the line-based boilerplate stripper failed; fixed to substring-strip. Post-sign-off UI feedback applied + re-verified live (full-height input/output panes + visible "INDENT" label). Signed off 2026-06-02. Artifacts: `07-VERIFICATION.md`, `07-REVIEW.md`, `07-UI-REVIEW.md`, `07-HUMAN-UAT.md`.

## Milestone Summary

**Decimal Phases:** None (Phases 7 and 8 are sequential integer phases).

**Key Decisions:**

- **Zero new runtime dependencies** — both formatters built on native `JSON` / `DOMParser` / `XMLSerializer`; the zero-dep ethos that defined v1.0 held into v1.1.
- **Shared two-pane `FormatterView`** — JSON and XML reuse one resizable side-by-side presentational component (read-only output pane, shared toolbar with conditional sort-keys), promoted from the protobuf `ResizableSplit`. No layout duplication.
- **Pure logic in `src/lib/format/`** — `formatJson`/`formatXml` are pure, engine-portable, independently unit-tested (TDD) and free of React, keeping the GUI thin and a future CLI (backlog 999.4) reachable.
- **Registry-only tool registration** — both tools appear in sidebar/palette/router solely by appending to the `TOOLS` array (single control plane), no other wiring.
- **StatusBar `byteCount` made opt-in via a minimal additive API** — optional prop + `typeof byteCount === "number"` guard, no discriminated-type machinery; keep/drop verified against the complete set of callers (which is why Phase 8 depended on Phase 7).
- **Sequential phase ordering (8 depends on 7)** — the cleanup was deliberately scheduled after the Formatters consumed `StatusBar`, so the keep/drop split was decided against real callers, not anticipated ones.

**Issues Resolved:**

- XML prettify originally dropped the `<?xml?>` declaration + doc-level comments/PIs (code-review WR-01) — fixed.
- Timing chip measured a React state setter rather than the format pass (WR-02) — fixed.
- First-match `indexOf` mislocated the V8 JSON error offset (WR-03) — fixed.
- Real-WebKit `<parsererror>` concatenates text with no newlines, breaking the line-based boilerplate stripper (caught only by the live e2e gate, not unit tests) — fixed to substring-strip; unit test updated to the real captured shape.

**Issues Deferred:**

- **FormatterView narrow-width vertical stacking (UX-05)** — confirmed NOT a WCAG-AA blocker by the Phase-7 UI audit; carried forward as non-blocking polish.

**Technical Debt Incurred:**

- None material. The `08-HUMAN-UAT.md` artifact persists at `status: partial` per the GSD workflow (human-approved at the boundary; `/gsd-verify-work 8` would formally close it) — bookkeeping only, not a code-level debt.

**Carry-forwards from v1.0 (still open, not v1.1 regressions):**

- Gatekeeper-clean notarisation DEFERRED post-Apple-Developer enrolment (D-02) — credentials-only `APPLE_*` env flip per `docs/RELEASE.md`.
- NAT-01 configurable global summon hotkey PARKED (G-05-1) — seam intact for a future Settings phase.
- 3 minor a11y polish follow-ups from the updater UI review.
- Backlog: Protobuf decimal-byte-array input mode; 999.1 more-tools / SQL formatter; 999.4 DevTools CLI.
