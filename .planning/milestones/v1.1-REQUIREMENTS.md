# Requirements: DevTools

**Defined:** 2026-06-02
**Milestone:** v1.1 Formatters
**Core Value:** Paste an unknown blob → get a usable, explorable interpretation in under 2 seconds, offline, without the mouse.

## v1.1 Requirements

Requirements for the Formatters milestone. Each maps to exactly one roadmap phase.
Design spec: `docs/superpowers/specs/2026-06-02-json-xml-formatters-design.md`.

All requirements inherited the standing product constraints: offline/no-network, paste-instant
(<2s), keyboard-driven, registry-driven, WCAG-AA, layout-agnostic, zero new runtime deps, and the
hero decoder (`src/lib/protobuf/decoder.ts`) + its 19 tests stayed untouched.

### Formatters (FMT)

- [x] **FMT-01**: User can paste JSON and see it validated instantly; on invalid input the error is shown as line:col + message — ✓ v1.1 (Phase 7)
- [x] **FMT-02**: User can prettify JSON with a selectable indent (2 spaces / 4 spaces / tab) — ✓ v1.1 (Phase 7)
- [x] **FMT-03**: User can minify (compress) JSON to a single line — ✓ v1.1 (Phase 7)
- [x] **FMT-04**: User can sort JSON object keys recursively (array order preserved) — ✓ v1.1 (Phase 7)
- [x] **FMT-05**: User can paste XML and see it validated for well-formedness instantly; on invalid input the parser error is surfaced — ✓ v1.1 (Phase 7)
- [x] **FMT-06**: User can prettify XML with a selectable indent, preserving comments, CDATA, attributes, and processing instructions (and the `<?xml?>` declaration) — ✓ v1.1 (Phase 7)
- [x] **FMT-07**: User can minify (compress) XML by stripping insignificant inter-element whitespace — ✓ v1.1 (Phase 7)
- [x] **FMT-08**: User can copy the formatted output via a visible, keyboard-focusable control (no hover-only) — ✓ v1.1 (Phase 7)

### UX Cleanup (UIX)

- [x] **UIX-01**: The status-bar size readout appears only where input/output size is meaningful (Base64/Hex/Bytes, Protobuf decoder, and the Formatters); it is removed from Hash, UUID/ULID, Unix Time, and JWT (status text only) — ✓ v1.1 (Phase 8)

## Outcomes & Notes

- **All 9 requirements validated and shipped** (FMT-01..08 + UIX-01). No requirements were dropped or materially adjusted during the milestone.
- **FMT-06 strengthened in flight:** the original wording covered comments/CDATA/attributes/PIs; a code-review finding (WR-01) surfaced that prettify was also dropping the `<?xml?>` declaration, so declaration preservation was added — a tightening, not a scope change.
- **Zero-new-dependency Out-of-Scope item held:** both formatters are native (`JSON`, `DOMParser`/`XMLSerializer`); no runtime dependency was added.

## Deferred (carried to backlog, not part of v1.1)

- **SQL formatter** — needs the `sql-formatter` library (only reformats, cannot lint). Parked in backlog 999.1.
- **JSON5 / JSONC** (comments, trailing commas) — strict JSON only for v1.1.
- **XML schema/XSD validation** — well-formedness only for v1.1.
- **DevTools CLI** — invoke tools from the command line. Backlog 999.4.
