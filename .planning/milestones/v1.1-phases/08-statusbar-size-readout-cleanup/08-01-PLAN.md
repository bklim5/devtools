<objective>
Make the shared `StatusBar` byte/size readout opt-in (UIX-01). Today `byteCount` is a
required prop and the size `<span aria-label="byte count">` ALWAYS renders. After this plan,
`byteCount` is optional: the size span renders only when a caller passes a number; absent →
no size text, everything else (state label, encoding chip, error, timing) unchanged.

Then remove the `byteCount` prop from the four drop tools (Hash, UUID/ULID, Unix Time, JWT)
so their size readout disappears, and lock the keep/drop split with present/absent assertions
across all affected tool tests.

Purpose: The size readout is only meaningful where input/output size matters
(Base64/Hex/Bytes, the Protobuf decoder, the Formatters). On Hash/UUID/Unix Time/JWT it is
either a meaningless `0` or (for Hash) a real-but-pointless byte count of the hashed input.
Opt-in keeps the StatusBar honest.

Output: An optional-`byteCount` `StatusBar`, four drop tools with the prop removed, and tests
that assert present-where-kept / absent-where-dropped via the `aria-label="byte count"` span.

Scope guardrails (binding): the hero decoder `src/lib/protobuf/decoder.ts` and its 19 tests
stay byte-for-byte untouched; NO new runtime dependencies; NO discriminated/conditional prop
type (keep StatusBar presentational and primitive-only); NO other StatusBar behavior changes
(state label, error text, timing, encoding chip all unchanged); NO richer status-left text for
the drop tools. Tools import `src/lib/platform/`, never `@tauri-apps/*`.
</objective>

This phase is a purely presentational refactor of an offline desktop app: it makes one
existing rendered text element (the `aria-label="byte count"` size span in the shared
`StatusBar`) conditional, and removes the prop that feeds it from four tools. It introduces
**no new input handling, no network calls, no authentication, no secrets, no persistence,
and no new data-flow surface**. All byte-count values are derived from already-trusted,
already-rendered local clipboard/textarea input that the existing tools handle today; nothing
about how that input is parsed or trusted changes.

| (none new) | No trust boundary is added, removed, or crossed by this change. |
