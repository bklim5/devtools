## v1 Requirements — all complete ✅

### Tool Reordering (REORD)

- [x] **REORD-01**: User can drag a sidebar tool (via a grip handle shown on hover/focus) to a new position, with a clear drop indicator showing where it will land. Native drag (no dnd library). — Plan 16-02 (handle-initiated native HTML5 drag + neutral `tx-2` insertion line + end-of-list drop zone). **Validated.**
- [x] **REORD-02**: A plain click on a tool still navigates to that tool — dragging never triggers from a normal click (drag is handle-initiated). — Plan 16-02 (only the grip handle is `draggable`; `NavLink` click navigates; proven on the real WKWebView). **Validated.**
- [x] **REORD-03**: User can reorder a keyboard-focused tool with `Alt+↑` / `Alt+↓`, moving it one slot per press, without introducing a roving arrow-key navigation system (plain arrows stay unbound in the sidebar). — Plan 16-02 (Alt+arrow only; plain arrows fall through). **Validated.**
- [x] **REORD-04**: Each reorder is announced to assistive technology via an `aria-live` region (e.g. "Moved Cron to position 3 of 11"), and the moved tool retains keyboard focus. (WCAG-AA.) — Plan 16-02 (`aria-live="polite"` + ref-map focus retention; gsd-ui-review WCAG-AA 22/24). **Validated.**
- [x] **REORD-05**: The user's custom tool order persists across app restarts (stored as a `toolOrder` array of tool IDs via the existing preferences/store seam; the registry stays the canonical source). — Plan 16-01 (toolOrder field + coerceToolOrder + setToolOrder, round-tripped through the prefs seam; proven across reload + real restart). **Validated.**
- [x] **REORD-06**: A tool shipped in a later app version appears at the bottom of an existing custom order; an order referencing an unknown or removed tool ID degrades gracefully (no crash, no missing or duplicated tools). — Plan 16-01 (reconcileToolOrder: append-new + drop-unknown + de-dupe, output always a registry permutation). **Validated.**
- [x] **REORD-07**: User can reset the sidebar to the default (registry) order. — Plan 16-02 (right-click + keyboard-reachable Shift+F10 "Reset order" sets `toolOrder=[]`; backbone in 16-01). **Validated.**

## Out of Scope (held)

- **Reordering the ⌘K command palette or affecting routing** — only the sidebar render order changes; palette and router stayed order-agnostic. ✅ held.
- **Grouping/sections in the sidebar** — the list stays flat. ✅ held.
- **A dnd/animation library** — zero-new-runtime-deps held; native drag events only. ✅ held.
- **Cross-device order sync** — local-only, offline. ✅ held.
