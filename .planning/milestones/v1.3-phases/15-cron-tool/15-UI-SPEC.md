# Phase 15 — UI Design Contract

> Visual and interaction contract for the Cron tool (CRON-01..11, the 12th tool / 6th and final tool of v1.3).
> Generated by gsd-ui-researcher; verified by gsd-ui-checker.
> Pre-populated almost entirely from upstream: `15-CONTEXT.md` (locked decisions), `15-RESEARCH.md`,
> `REQUIREMENTS.md` (CRON-01..11), and the established LOCKED design system
> (`src/index.css` `@theme`, `design/DevTools Mockup.html`). Tokens are NOT re-invented — this tool
> inherits the exact same system the 11 shipped tools use. Where the Cron tool needs something the
> others don't (a description headline, a next-run list, the `@reboot` / "never" / unsupported-token
> states), it is specified in the EXISTING design language, mirroring the Regex (Phase 14), URL
> (Phase 13) and Unix Time tools precisely.

## Spacing Scale

The shared shell already uses a 4-based scale via Tailwind utilities. This tool reuses it verbatim;
the sibling `RegexTool.tsx` / `UrlTool.tsx` / `UnixTimeTool.tsx` are the precedents (`gap-4`, `p-4`,
`p-3`, `px-3 py-1.5`, `gap-2`, `gap-1.5`, `gap-3`).

| Token | Value | Usage in this tool |
|-------|-------|--------------------|
| xs | 4px | `gap-1` between inline meta items (e.g. run index `#1` ↔ relative-time chip); intra-row spacing |
| sm | 8px | `gap-2` label→field, section-to-section within a block |
| md | 12px | `gap-3` / `p-3` row internal padding + input padding (the established intra-row rhythm — matches UrlTool readout rows, RegexTool fields) |
| lg | 16px | `gap-4` / `p-4` section gaps + outer tool padding (the standard tool frame `flex flex-col gap-4 overflow-auto p-4`) |
| xl | 24px | reserved for a major vertical break between the description headline and the run list, if visual separation is wanted |

Row vertical padding for the next-run list rows: `px-3 py-1.5` (the URL readout-row value, `1.5` = 6px)
OR `px-3 py-2` (the Unix Time `OutputRow` value, `2` = 8px). Use **`px-3 py-2`** for the run-list rows —
they pair a two-line stack (zone-local datetime + relative caption) like Unix Time's `OutputRow`, whose
`py-2` is the right precedent.

Exceptions:
- **12px (`p-3`, `gap-3`) is the established intra-row rhythm** in this codebase and is retained for visual
  consistency rather than snapping to 16px. This is the shipped house norm across the URL / Protobuf /
  Base64 / Regex / Unix Time tools, not a violation.
- Touch/click targets: the input and every `CopyButton` inherit the shared primitives' sizing
  (focus-visible ring, keyboard operable) — already WCAG-AA.

## Typography

Inherited from the design system (`design/DevTools Mockup.html` + `src/index.css`). UI chrome is
IBM Plex Sans; **the cron expression input, every run-time value, and the IANA zone label are
JetBrains Mono** (`--font-mono`) because each character is load-bearing / a literal value to copy.
The human-readable **description headline is sans** (it is prose, not a code value).

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Description headline | 16px | 600 (semibold) | ~1.3 | sans | The prominent parsed-expression description (CRON-01), e.g. "At 09:00, Monday through Friday." The one larger type size — it is the tool's primary readout and earns visual weight. |
| Section label | 12px | 600 (semibold) | ~1.2 | sans | Uppercase tracking-wide field labels ("CRON EXPRESSION", "NEXT RUNS", "DESCRIPTION") — mirrors UrlTool / RegexTool labels exactly. |
| Body / caption | 12px | 400 (regular) | 1.4 | sans | Empty-state hint, error text, "no upcoming runs" message, the zone-context caption, the relative-time caption under each run ("in 3 hours"). |
| Run index / meta | 11px | 400 (regular) | ~1.4 | sans/mono | The `#1`..`#5` run ordinal and `@N` / `→` meta glyphs render `text-tx-3` neutral (mirrors RegexTool `@ {index}` and UrlTool key→value meta). |
| Code / value | 13px | 400 (regular) | 1.5 | mono | The cron expression `<input>`, each next-run datetime string, the IANA zone label. |

Weights used: exactly **2** — regular (400) for content/captions, semibold (600) for labels and the
description headline. (IBM Plex 500/700 are loaded in the system but this tool uses only 400/600,
matching RegexTool/UrlTool.)

Type sizes used: **3** core sizes — 16px (description headline, the single emphasis size), 12px
(labels + captions), 13px (mono values) — with an 11px meta tier inherited from the shipped tools for
row ordinals/copy buttons (same as RegexTool `@ {index}` / `CopyButton`). This matches the established
3-to-4-size discipline; no new size is introduced.

Line heights:
- 1.5 for the mono value surfaces (the input and run-time strings) — the shell norm for code.
- ~1.2–1.4 for sans labels/captions; ~1.3 for the slightly larger description headline.

## Color

60/30/10 inherited from the system `@theme` (`src/index.css`). The window is dark; accent is strictly
reserved (the project's standing rule: **accent = selected/active only**, never "all interactive"). The
hero Protobuf rule "`#N` numbers neutral, accent = selected only" generalizes here: the run ordinals
`#1`..`#5` are **neutral**, not accent (they enumerate, they do not select).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-bg-app` `#0a0b0d` (radial-gradient base) + `--color-input-bg` `#0d0f13` | Window background; the cron expression input field surface |
| Secondary (30%) | `--color-pane` `#14161b` / `--color-card`/`--color-panel` `#181b21` / `--color-input-bg/40` tint + `--color-bd` `rgba(255,255,255,.07)` borders | Tool frame, run-list rows, description block surface, dividers |
| Accent (10%) | `--color-accent` `#5b9bf8`, with `--color-accent-soft` (15% tint) + `--color-accent-line` (75%) | See reserved list below |
| Destructive / error | `--color-bad` `#f0876b` | Inline error text only (invalid expression; `W`/`#`/`LW` unsupported) — see note |
| Positive | `--color-ok` `#34d399` | Reserved — NOT required this phase (no success-state surface needed) |

**Accent (`#5b9bf8`) reserved for — explicit list (nothing else may use accent):**
1. **Focus-visible ring** on the input and every `CopyButton`/chip (`focus-visible:ring-accent`) — the
   system-wide a11y affordance, inherited unchanged.
2. **Copy-confirmed** state on `CopyButton` (the momentary "Copied" → `border-accent-line text-accent`),
   inherited unchanged.
3. **Active segment** of any toggle, IF one is added (e.g. a 5/6-field hint or rows/cards toggle) — via
   `SegmentedControl` (`border-accent-line bg-accent-soft text-accent`). None is required by the
   requirements; only add a toggle if implementation needs one (Claude's discretion), and if so it
   inherits this accent-on-active idiom.

**NOT accent (must stay neutral):**
- Run **ordinals** `#1`..`#5` render `text-tx-3` (neutral) — mirrors the hero's "`#N` numbers neutral"
  rule and RegexTool's neutral `$N` group numbers / `@ {index}`. Accent is for *selection*, not
  *enumeration*.
- The **IANA zone label** (e.g. `Asia/Singapore`) renders `text-tx-2` (neutral, mono) — it is metadata,
  not a selected state.
- The **description headline** renders `text-tx` (the brightest neutral) — emphasis comes from size +
  weight, not color.
- The **`@reboot` "run at startup"** banner is a NEUTRAL informational state (`text-tx-2`), explicitly
  NOT accent and NOT error (see Copywriting).
- The **"no upcoming runs"** state is a NEUTRAL/calm message (`text-tx-3`), explicitly NOT `text-bad`
  (it is a valid result, not an error — see Copywriting + RESEARCH CRON-08).

**Destructive note:** This tool has **no destructive action** — it is a pure, disposable string→readout
transform (paste, read, copy; the field is instantly re-editable). `--color-bad` is used ONLY as the
**error text color** (`role="alert"`, CRON-11 + the unsupported-token reject), never for a destructive
button. No confirmation dialog exists in this phase.

## Copywriting Contract

All copy below is prescriptive. Empty / error / "never" / `@reboot` discipline mirrors
Base64/Protobuf/URL/Regex error-state conventions. Note: this is a **paste-instant** tool — there is
**no "Parse"/"Compute" CTA button** (the description + next runs recompute live on input, the project's
no-decode-button wedge). There is no primary action button at all; the only affordances are the input
and the per-row copy buttons.

| Element | Copy |
|---------|------|
| Primary CTA | **None.** Paste-instant; no button. (The closest "action" is `CopyButton` on each run time / the description — see below.) |
| Field label | **`CRON EXPRESSION`** (uppercase semibold label, UrlTool/RegexTool style). |
| Input placeholder | **`0 9 * * 1-5   ·   @daily   ·   0 0 L * *`** (shows 5-field, a macro, and an `L` form so the supported grammar is discoverable; 24-hour examples only). |
| Description block label | **`DESCRIPTION`** (uppercase semibold). |
| Description headline (normal) | The hand-rolled 24-hour description string, verbatim from the core, e.g. **"At 09:00, Monday through Friday."** / **"Every 15 minutes."** Always 24-hour time (locked, CRON-01). Rendered as escaped React text (`text-tx`, sans 16px/600). A `CopyButton` may sit beside it (optional, low priority). |
| Next-runs block label | **`NEXT RUNS`** (uppercase semibold). When runs exist, the heading reads **`NEXT RUNS (5)`** mirroring UrlTool's `Query ({n})` / RegexTool's `Matches ({n})`. |
| Zone caption (under NEXT RUNS) | Neutral caption (sans 12px `text-tx-3`): **"Local time · {IANA zone}"**, e.g. **"Local time · Asia/Singapore"** — the system zone (locked: system-zone-only, no picker). Satisfies CRON-05's "IANA timezone label" requirement once, above the list; each row need not repeat the full zone. |
| Each run row | Two lines: a mono datetime value (`text-tx`, 13px) — locale-formatted 24-hour local time via `Intl.DateTimeFormat` (`hourCycle:"h23"`) — and a neutral relative caption beneath (sans 12px `text-tx-3`), e.g. **"in 3 hours"** / **"tomorrow"** / **"in 2 days"**. Each row carries a visible, focusable `CopyButton` (no hover-only). Row ordinal **`#1`..`#5`** in `text-tx-3` neutral. |
| Empty state (no input) | Neutral hint, NOT an error: **"Paste a cron expression to see its description and next 5 run times."** No error chrome, no `role="alert"`. (Mirrors UrlTool "Paste an absolute URL…" / RegexTool empty hint.) |
| `@reboot` state (CRON-09) | A NEUTRAL informational state, explicitly distinct from BOTH the normal run list AND any error. Description headline reads **"At startup — runs once when the scheduler starts."** Then, in place of the NEXT RUNS list, a single calm caption (sans 12px `text-tx-2`, NOT `text-bad`): **"No scheduled runs — `@reboot` fires only on startup, so there is no upcoming time to compute."** The `NEXT RUNS` heading + zone caption are OMITTED in this state (there is no list to label). |
| "Never fires" / impossible expression (CRON-08) | A calm, NON-error message, NOT `text-bad`: under the `NEXT RUNS` heading, a neutral line (sans 12px `text-tx-3`): **"No upcoming runs in the next 5 years — this expression may never fire (e.g. February 30)."** The DESCRIPTION headline still renders normally above it (the expression parsed fine; it simply has no future match). This is visually the same family as RegexTool's "No matches." / UrlTool's "No query parameters." — a calm empty-of-results state, never error chrome. |
| Error state — invalid expression (CRON-11) | Inline `role="alert"` in `text-bad`, rest of view intact, no throw. Copy = the core's specific named-token message (mirror `url.ts`/`bytes.ts` named errors), e.g. **"Out of range: minute "75" must be 0–59."** / **"Wrong field count: expected 5 or 6 fields, got 4."** / **"Unparseable token: "abc" in day-of-week."** Name the bad token AND the valid range/expectation. Both the description headline and the run list are suppressed in this state; `aria-invalid` set on the input. |
| Error state — unsupported token `W` / `#` / `LW` (locked reject, CRON-F1) | Same inline `role="alert"` `text-bad` error path (consistent with CRON-11 — NOT a special half-feature state). Copy: **"`W`, `#` and `LW` aren't supported yet — try a standard field, range, step, list, or `L` / `nL` / `L-n`."** Names the unsupported token AND the supported alternatives. |
| Unmatched / empty sub-value (if any) | Muted em-dash **`—`** in `text-tx-3` (the shipped placeholder idiom — UrlTool `Value` empty / RegexTool optional-group). Likely unused here, but reserved for consistency. |
| Destructive confirmation | None — no destructive action in this phase (see Color §Destructive note). |

**Copy tone:** terse, developer-direct, 24-hour time throughout — consistent with the shipped tools. No
marketing voice. Error messages name the problem AND the next step / valid range (the project's
error-state discipline). The `@reboot` and "never" states are calm and explanatory, never alarming —
they are valid outcomes, not failures.

> Not a template section, but load-bearing for the planner/executor. Each maps to a requirement.

| Component | Built from | Requirement | Notes |

| Error / unsupported-token banner | Inline `role="alert"` `text-bad` `text-[12px]`, rest of view suppressed | CRON-11 + reject `W`/`#`/`LW` | error-as-value; never a throw. `aria-invalid` on input. |
| Copy affordances | Shared `CopyButton` (`@/components/CopyButton` or the local visible-button pattern) → `platform.clipboard` seam + `useCopyFeedback` | cross-cutting | Always-visible, focusable; NO hover-only copy (CLAUDE.md constraint). On each run row; optionally on the description. |

**Layout:** ONE combined scrolling view, NO mode switch by default — input → description → next runs,
all live at once (mirrors RegexTool's single-view structure). Outer frame is the standard
`flex min-w-0 flex-1 flex-col` with an inner `flex min-w-0 flex-1 flex-col gap-4 overflow-auto p-4`.
Layout-agnostic (responsive Tailwind, `min-w-0`, no fixed widths; layout chrome lives in the shell).
**StatusBar / byteCount: OMIT** — cron is not a byte transform (Phase 8 opt-in precedent; RegexTool
also omits it).

**Cards-vs-rows idiom:** the run list uses the established **labeled-row** form (Unix Time `OutputRow`),
NOT the Protobuf cards/rows toggle — there is one flat list of 5 homogeneous runs, so a single row
style is correct and no rows/cards toggle is warranted. (The Protobuf cards/rows toggle exists because
protobuf fields are heterogeneous and nest; cron runs are not.)

## Accessibility Notes (WCAG-AA, phase-boundary gate)

- Phase boundary requires a passing `gsd-ui-review` WCAG-AA audit + `tauri build` human walkthrough (the
  v1.3 real-WKWebView gate applies — these are UI tools).
- All accent-on-tint text combinations are pre-cleared: `--color-accent #5b9bf8` was deliberately
  brightened to clear AA on `accent-soft`; `--color-tx-3 #868b95` was raised to clear AA for 11–12.5px
  body text (`src/index.css` comments, 03-UI-REVIEW). The neutral run ordinals, zone label, relative
  captions, and "never"/`@reboot` text all use the AA-cleared `text-tx-3`/`text-tx-2`/`text-tx` ramp.
- The input, every `CopyButton`, and any toggle are keyboard-operable with `focus-visible:ring-accent`
  (inherited from the shared primitives / shipped patterns).
- **Copy is never hover-gated** (project rule, CLAUDE.md) — every run-time `CopyButton` is a visible,
  focusable button.
- Error AND unsupported-token states use `role="alert"` (or `aria-live="polite"` per the Unix Time
  field-error precedent) so they are announced; `aria-invalid` is set on the input on error.
- The `@reboot` and "no upcoming runs" states are NEUTRAL informational text (not `role="alert"`,
  not `text-bad`) — they are valid results and must not be announced as errors.
- 24-hour time is used in BOTH the description headline AND every run-time value (locked, CRON-01;
  `hourCycle:"h23"` so midnight never renders as `24:00`, per RESEARCH Pitfall 8).
