<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pasted text → UrlTool view | Untrusted pasted URL / arbitrary string rendered in the DOM. Offline, no network, no backend. |
| view → platform.clipboard | Copy writes decoded values (incl. password, D-09) to the OS clipboard on explicit user action only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering / XSS | Rendering decoded URL parts + query values | mitigate | All values rendered as React text children (default escaping); `dangerouslySetInnerHTML` is FORBIDDEN and asserted absent (acceptance criteria `grep -L`). A pasted `<script>`/`javascript:` value is shown as inert text, never executed. |
| T-13-05 | Denial of Service | Malformed URL / %zz crashing the view | mitigate | All parse/decode flows through 13-01's error-as-value helpers; the view renders `role="alert"` inline and keeps the rest intact (D-13/D-14). e2e exercises the relative-URL path on the real WKWebView. |
| T-13-06 | Information Disclosure | password copied to OS clipboard (D-09) | accept | Plaintext password display + copy is an explicit user decision for auth-URL debugging; clipboard write only on user click, value never logged/persisted by the tool. Offline, local-only; user already holds the URL. |

No high-severity threats: pure-frontend, offline, no network/auth/backend/persistence, no code-eval. Surface = DOM-injection (mitigated by React escaping + no dangerouslySetInnerHTML) and throw-safety (mitigated by error-as-value), plus the consciously-accepted plaintext password.
</threat_model>
